def transform(s: str) -> str:
    res = []
    i = 0
    n = len(s)
    while i < n:
        if s[i] == ' ':
            # copy spaces as they are
            res.append(s[i])
            i += 1
        else:
            # start of a word
            j = i
            while j < n and s[j] != ' ':
                j += 1
            word = s[i:j]
            if len(word) > 2:
                word = word[0].upper() + word[1:].lower()
            # else keep original word unchanged
            res.append(word)
            i = j
    return ''.join(res)

**Solution Explanation**

For a string to be an *accordion* it must have the form  

```
[ : (|…|) : ]
```

i.e.

* a `'['` character,
* later a `':'`,
* later a `']'` character,
* before that `']'` a `':'`,
* between the two colons any number (possibly zero) of `'|'`.

The length of such an accordion is  

```
4  (the two brackets and the two colons)
+ number of '|' characters between the two colons
```

The task is to delete characters from the given string `s` (keeping the order)
so that the remaining characters form an accordion, and to maximise its length.
If it is impossible we output `-1`.

--------------------------------------------------------------------

#### Observations
* To maximise the length we want the *earliest* possible opening `'['` and the
  *latest* possible closing `']'`.  
  Any later `'['` would only shrink the interval we can use,
  any earlier `']'` would also shrink it.
* After fixing the positions of `'['` and `']'` we must pick a colon after the
  `'['` and a colon before the `']'`.  
  The earliest colon after `'['` and the latest colon before `']'` give the
  widest interval for `'|'` characters, therefore the maximal possible length.
* If any of these four characters does not exist in the required order,
  an accordion cannot be formed.

--------------------------------------------------------------------

#### Algorithm
```
1. Scan from left to right.
   Find the first '['. After it, find the first ':'.
   Let its index be left_colon. If not found → impossible.

2. Scan from right to left.
   Find the first ']'. Before it, find the first ':'.
   Let its index be right_colon. If not found → impossible.

3. If left_colon >= right_colon → impossible (colons cross).

4. Count the character '|' in s[left_colon+1 : right_colon].
   Answer = 4 + that count.

5. Output answer.
```

All scans are linear, the whole algorithm runs in `O(|s|)` time
and `O(1)` additional memory.

--------------------------------------------------------------------

#### Correctness Proof  

We prove that the algorithm outputs `-1` iff no accordion can be obtained,
otherwise it outputs the maximum possible length.

---

**Lemma 1**  
If an accordion can be obtained from `s`, there exists one whose `'['`
is the first `'['` in `s` and whose `']'` is the last `']'` in `s`.

*Proof.*  
Take any feasible accordion. Its `'['` appears at some position `i`.
If there is an earlier `'['` in `s` (position `< i`), we can replace the
chosen `'['` by that earlier one; all later characters keep their relative order,
so the resulting subsequence is still a valid accordion and is not longer.
Similarly, we can replace the chosen `']'` by the last `']'` in `s`. ∎



**Lemma 2**  
For fixed positions of `'['` and `']'`, the longest possible accordion uses
the earliest `':'` after `'['` and the latest `':'` before `']'`.

*Proof.*  
The two colons must lie strictly between the brackets.
Choosing a later left colon shortens the interval where `'|'` may appear,
so cannot increase the number of `'|'`. The same holds for the right colon. ∎



**Lemma 3**  
If the algorithm reports “impossible”, then no accordion can be formed.

*Proof.*  
The algorithm fails only in three cases:

1. No `':'` after the first `'['`. Any accordion needs such a colon,
   otherwise the pattern `[ : … : ]` cannot be completed.
2. No `':'` before the last `']'`. Symmetric argument.
3. The left colon is not strictly left of the right colon.
   Then any two colons satisfying the pattern would have to cross,
   which is impossible.

Thus in each case the required pattern cannot be realized. ∎



**Lemma 4**  
If the algorithm produces a length `L`, then an accordion of length `L`
can be obtained from `s`.

*Proof.*  
The algorithm selects:
* the first `'['`,
* the earliest `':'` after it,
* the latest `':'` before the last `']'`,
* the last `']'`.

All these characters appear in `s` in the required order,
and the algorithm keeps every `'|'` between the two chosen colons.
Consequently the subsequence consisting of those characters is exactly an
accordion whose length is `4 + (# of kept '|') = L`. ∎



**Lemma 5**  
No accordion obtainable from `s` is longer than the length `L` returned by
the algorithm.

*Proof.*  
Consider any feasible accordion.
By Lemma&nbsp;1 we may assume it uses the first `'['` and the last `']'`.
By Lemma&nbsp;2 we may assume it uses the earliest possible left colon
and the latest possible right colon.
Therefore the set of `'|'` characters it can keep is a subset of the `'|'`
between the two colons chosen by the algorithm.
Hence its length cannot exceed `4 + (that subset size) ≤ L`. ∎



**Theorem**  
The algorithm outputs `-1` iff no accordion can be obtained;
otherwise it outputs the maximum possible length of an accordion that can be
obtained from `s`.

*Proof.*  
If the algorithm outputs `-1`, by Lemma&nbsp;3 no accordion exists.
If it outputs a number `L`, Lemma&nbsp;4 shows an accordion of length `L`
exists, and Lemma&nbsp;5 shows no longer accordion exists.
Thus `L` is the optimum. ∎



--------------------------------------------------------------------

#### Complexity Analysis
The string is scanned twice, each in `O(n)` time (`n = |s| ≤ 5·10⁵`).
Counting `'|'` is another linear pass over the interval.
Memory usage is `O(1)`.

import math

def getAllPrimeFactors(n: int):
    """Return list of prime factors of n in ascending order.
    Handles special cases as described."""
    if n <= 0:
        return []
    if n == 1:
        return [1]

    factors = []
    # factor out 2s
    while n % 2 == 0:
        factors.append(2)
        n //= 2
    # odd factors
    f = 3
    max_f = int(math.isqrt(n)) + 1
    while f <= max_f and n > 1:
        while n % f == 0:
            factors.append(f)
            n //= f
            max_f = int(math.isqrt(n)) + 1
        f += 2
    if n > 1:
        factors.append(n)
    return factors

    n = len(s)

        print("")
        return

    runs = []  # list of (char, length, start_index)
    i = 0
    while i < n:
        ch = s[i]
        j = i
        while j < n and s[j] == ch:
            j += 1

    best_len = 0
    best_start = -1

    for k in range(len(runs) - 1):

        c2, l2, st2 = runs[k + 1]

            cur_len = l1 + l2
            if cur_len > best_len or (cur_len == best_len and st1 < best_start):
                best_len = cur_len
                best_start = st1

    if best_len == 0:
        print("")
    else:
        print(s[best_start:best_start + best_len])

* **exception rules** (they have priority)

| original `ch` | shift |
|---------------|-------|
| `c` or `o`    | –1    |

| kind of letter | shift |
|----------------|-------|
| vowel (`a e i o u`) | –5 |
| consonant (all other letters) | +9 |

The shift is performed on the circular alphabet `a … z`.  
After the shift we obtain a candidate character `new_ch`.  
If `new_ch` is one of the letters `c, o, d, e` we **undo** the change and keep
the original character.

The algorithm is linear in the length of the string.

    def process_token(token: str) -> str:
        # If token is only whitespace, return as is
        if token.isspace():
            return token

        # Separate trailing punctuation
        core = token
        suffix = ''
        while core and core[-1] in punct_set:
            suffix = core[-1] + suffix
            core = core[:-1]

        # If after removing punctuation nothing remains, just return token
        if not core:
            return token

        t_count = sum(1 for ch in core if ch == 't' or ch == 'T')

        # Rule 1: reverse
        if length > 6 or t_count >= 2:

        if length == 2 or (suffix and suffix[0] == ','):
            new_core = core.upper()
            return new_core + suffix

        # Rule 3: replace single character with "0"
        if length == 1:
            new_core = '0'
            return new_core + suffix

        # No transformation
        return core + suffix

    out_parts = []
    for line in lines:
        # Split line into words and spaces preserving them
        tokens = re.findall(r'\S+|\s+', line)
        transformed = [process_token(tok) for tok in tokens]

**Solution Explanation**

For an *accordion* we need a subsequence  

```
'['  ':'  (any number of '|')  ':'  ']'
```

The order of the characters must be preserved, we may delete other characters.
If the two colons are at positions `c1 < c2`, the length of the obtained
accordion is  

```
4  +  (number of '|' between c1 and c2)
```

So we have to maximise the number of `'|'` between two suitable colons.

--------------------------------------------------------------------

#### 1.   When can a colon be used ?

* a colon `c` can be the **left** colon iff there is a `'['` somewhere
  before it;
* a colon `c` can be the **right** colon iff there is a `']'` somewhere
  after it.

If we choose the *earliest* possible left colon and the *latest* possible
right colon, the interval `[c1 , c2]` becomes as wide as possible,
therefore it contains the maximal possible amount of `'|'`.
Any other admissible pair of colons lies inside this interval and can
contain **not more** `'|'`.

Thus we only have to find

```
c1 = first ':' that has a '[' before it
c2 = last  ':' that has a ']' after it
```

If `c1` does not exist, or `c2` does not exist, or `c1 >= c2`,
forming an accordion is impossible.

--------------------------------------------------------------------

#### 2.   Computing the needed positions

One left‑to‑right scan:

```
found_left_bracket = False
for i = 0 .. n-1
        if s[i] == '[' : found_left_bracket = True
        if s[i] == ':' and found_left_bracket:
                c1 = i               # first suitable left colon
                break
```

One right‑to‑left scan:

```
found_right_bracket = False
for i = n-1 .. 0
        if s[i] == ']' : found_right_bracket = True
        if s[i] == ':' and found_right_bracket:
                c2 = i               # last suitable right colon
                break
```

Both scans are `O(n)`.

--------------------------------------------------------------------

#### 3.   Counting `'|'` between the two colons

If `c1 < c2` we simply count `'|'` in the substring `s[c1+1 : c2]`
(again linear, but only once).

The answer is  

```
answer = 4 + count_of_pipes
```

If the conditions are not satisfied we output `-1`.

--------------------------------------------------------------------

#### 4.   Correctness Proof  

We prove that the algorithm outputs `-1` iff no accordion can be formed,
otherwise it outputs the maximum possible length.

---

##### Lemma 1  
`c1` (found by the left‑to‑right scan) is the earliest colon that can be
used as the left colon of an accordion.

**Proof.**  
The scan stops at the first position where a colon is seen **and**
a `'['` has already been seen.  
Any earlier colon either appears before any `'['` (so it cannot be left
colon) or does not exist. ∎



##### Lemma 2  
`c2` (found by the right‑to‑left scan) is the latest colon that can be
used as the right colon of an accordion.

**Proof.**  
Symmetric to Lemma&nbsp;1: scanning from the end we stop at the first colon
that has a `']'` to its right, i.e. the rightmost colon with a following
closing bracket. No later colon can satisfy the condition. ∎



##### Lemma 3  
If an accordion exists, then `c1 < c2`.

**Proof.**  
Assume an accordion exists, i.e. there are indices  

```
i < j < k < l   with   s[i]='[', s[j]=':', s[k]=':', s[l]=']'.
```

From Lemma&nbsp;1, `c1` is the earliest colon after a `'['`, therefore
`c1 ≤ j`.  
From Lemma&nbsp;2, `c2` is the latest colon before a `']'`, therefore
`c2 ≥ k`.  
Since `j < k`, we have `c1 ≤ j < k ≤ c2`, thus `c1 < c2`. ∎



##### Lemma 4  
For any pair of admissible colons `x < y` (i.e. a `'['` before `x` and a
`']'` after `y`) the number of `'|'` between `x` and `y` is **not larger**
than the number of `'|'` between `c1` and `c2`.

**Proof.**  
From Lemma&nbsp;1, `c1` is the earliest admissible left colon, therefore
`c1 ≤ x`.  
From Lemma&nbsp;2, `c2` is the latest admissible right colon, therefore
`y ≤ c2`.  
Consequently the interval `[x , y]` is contained in `[c1 , c2]`.  
The count of `'|'` inside a sub‑interval cannot exceed the count inside the
whole interval. ∎



##### Lemma 5  
If `c1 < c2`, the string formed by taking `'['` before `c1`,
the colon at `c1`, all `'|'` between `c1` and `c2`,
the colon at `c2` and a `']'` after `c2` is a valid accordion whose
length equals `4 + (# of '|' between c1 and c2)`.

**Proof.**  
By construction there is a `'['` before `c1` (found during the left scan)
and a `']'` after `c2` (found during the right scan).  
The two chosen colons are at `c1` and `c2`.  
All characters kept are in the original order, therefore the subsequence
has exactly the required pattern. Its length is the four brackets/colons
plus the number of kept `'|'`. ∎



##### Theorem  
The algorithm outputs  

* `-1` iff no accordion can be obtained from `s`;
* otherwise the maximum possible length of an accordion obtainable from `s`.

**Proof.**  

*If the algorithm prints `-1`.*  
That happens when either `c1` or `c2` does not exist, or `c1 ≥ c2`.  
If any of them does not exist, there is no `'['` before a colon or no `']'`
after a colon, so an accordion is impossible.  
If `c1 ≥ c2`, by Lemma&nbsp;3 no accordion can exist. Hence `-1` is correct.

*If the algorithm prints a length `L`.*  
Then `c1 < c2`. By Lemma&nbsp;5 an accordion of length `L = 4 + cnt`
exists.  
Assume there is any other accordion with length `L' > L`. Its two colons
form an admissible pair `x < y`. By Lemma&nbsp;4 the number of `'|'`
between `x` and `y` is ≤ `cnt`, therefore `L' ≤ 4 + cnt = L`, contradiction.
Thus no longer accordion exists and `L` is maximal. ∎

def sieve(limit: int):
    """Return list of primes less than limit."""
    if limit <= 2:

    is_prime = bytearray(b'\x01') * limit
    is_prime[0:2] = b'\x00\x00'

        if is_prime[p]:
            step = p
            start = p * p

    return [i for i, val in enumerate(is_prime) if val]

def count_inversions(arr):
    # Helper function using merge sort
    def merge_sort(a):
        n = len(a)
        if n <= 1:
            return a, 0
        mid = n // 2
        left, inv_left = merge_sort(a[:mid])
        right, inv_right = merge_sort(a[mid:])
        merged = []
        i = j = 0
        inv_count = inv_left + inv_right
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                merged.append(left[i])
                i += 1
            else:
                merged.append(right[j])
                j += 1
                inv_count += len(left) - i  # all remaining in left are inversions
        merged.extend(left[i:])
        merged.extend(right[j:])
        return merged, inv_count

    _, total_inv = merge_sort(arr)
    return total_inv

The inner ring moves clockwise, i.e. one step changes the position from `x`
to `x‑1 (mod innerMax+1)`.  
For the outer ring we have the numbers `0 … outerMax` and it moves
anti‑clockwise, i.e. one step changes the position from `y` to `y+1 (mod
outerMax+1)`.

Both rings start at `0`.  
After `t` moves

* inner position   = `(-t) mod (innerMax+1)`
* outer position   = ` t   mod (outerMax+1)`

We have to find the smallest positive `t` for which the two positions are
equal.

--------------------------------------------------------------------

#### 1.   Reformulation

Let  

```
M = innerMax + 1          # number of inner positions
N = outerMax + 1          # number of outer positions
```

The required condition is  

```
(-t) mod M  =  t mod N                (1)
```

If `t` is a multiple of `M` then the left side is `0`.  
If additionally `t` is a multiple of `N` the right side is also `0`.  
Therefore every common multiple of `M` and `N` satisfies (1).  
The smallest such number is `lcm(M,N)` – this is one candidate.

The other possibility is that both remainders are non‑zero.
For a non‑zero remainder `r = t mod M ( 1 ≤ r < M )`

```
(-t) mod M = M - r
```

Equation (1) becomes  

```
t mod N = M - r                         (2)
```

Thus we need a pair of remainders whose sum is exactly `M`.

--------------------------------------------------------------------

#### 2.   Solving the “sum = M’’ case

Write `t = r + M·k` (`k ≥ 0`).  
Insert this into (2):

```
r + M·k ≡ M - r   (mod N)
M·k ≡ M - 2r      (mod N)                (3)
```

Let  

```
g = gcd(M, N)
```

`M·k ≡ M - 2r (mod N)` has a solution **iff** `g` divides `M‑2r`.
Because `g` always divides `M`, the condition reduces to  

```
g | 2r                                    (4)
```

So `r` must be a multiple of  

```
step = g / gcd(g, 2)          # the smallest positive integer satisfying (4)
```

All admissible remainders are `r = step, 2·step, … , M‑step`.

For a concrete `r` we solve (3).  
Divide the whole congruence by `g` :

```
(M/g)·k ≡ (M-2r)/g   (mod N/g)            (5)
```

`M/g` and `N/g` are coprime, therefore `M/g` has a modular inverse
`inv = (M/g)^{-1} (mod N/g)`.  
The smallest non‑negative solution of (5) is

```
k = inv * ((M-2r)/g)  (mod N/g)
```

Finally

```
t = r + M·k
```

Among all admissible `r` we keep the smallest positive `t`.  
The answer is the minimum of this value and `lcm(M,N)`.

The number of admissible remainders is at most `M/step`.  
`step` is at least `1` and at most `g`, therefore the loop is fast
(`M` ≤ 2·10^9 ⇒ at most a few million iterations, well inside limits).

#### 3.   Algorithm
```

N = outerMax + 1

g = gcd(M, N)

**Proof.**  
`t` multiple of `M` ⇒ `(-t) mod M = 0`.  
`t` multiple of `N` ⇒ `t mod N = 0`.  
Both positions are `0`, therefore equal. ∎



##### Lemma 2  
If the rings coincide after `t` moves and at least one of the remainders
is non‑zero, then `t mod M + t mod N = M`.

**Proof.**  
Let `r = t mod M` (`1 ≤ r < M`).  
`(-t) mod M = M - r`.  
Equality of the two positions gives `t mod N = M - r`.  
Rearranging yields `r + t mod N = M`. ∎



##### Lemma 3  
For a positive integer `r` (`1 ≤ r < M`) the congruence  
`M·k ≡ M - 2r (mod N)` has a solution `k` **iff** `g = gcd(M,N)` divides `2r`.

**Proof.**  
The congruence is equivalent to `M·k - (M-2r) = N·q` for some integer `q`.  
The left hand side is a linear combination of `M` and `2r`.  
A solution exists exactly when `g` divides the constant term `M-2r`.  
Since `g | M`, this is equivalent to `g | 2r`. ∎



##### Lemma 4  
All remainders `r` that can appear in a solution with non‑zero remainders are
exactly the multiples of `step = g / gcd(g,2)` and are smaller than `M`.

**Proof.**  
By Lemma&nbsp;3 we need `g | 2r`.  
Write `g = d·gcd(g,2)` with `d = g / gcd(g,2)`.  
Then `g | 2r` ⇔ `d | r`.  
Thus `r` must be a multiple of `d = step`.  
Conversely any such multiple satisfies the divisibility, therefore is
admissible. ∎



##### Lemma 5  
For a fixed admissible `r` the algorithm computes the smallest non‑negative
`k` that satisfies `M·k ≡ M - 2r (mod N)`.

**Proof.**  
Dividing the congruence by `g` (which is allowed because `g | M` and
`g | (M-2r)`) yields (5) where the modulus is `N/g` and the coefficient of
`k` is `M/g`, coprime to the modulus.  
The modular inverse `inv` of `M/g` exists, and the unique solution modulo
`N/g` is `k = inv·((M-2r)/g) (mod N/g)`.  
The algorithm computes exactly this value, which is the smallest
non‑negative solution. ∎



##### Lemma 6  
For every admissible `r` the value `t = r + M·k` produced by the algorithm
satisfies the coincidence condition (1) and has the smallest possible

**Proof.**

Adding `r` to both sides gives `r + M·k ≡ M - r (mod N)`.  
Since `r = t mod M`, the left side is `t`, the right side is `(-t) mod M`.

computed by the algorithm. ∎

**Proof.**

For a string to be an *accordion* it must have the form  

```
[ : |…| : ]
```

i.e.  

* an opening bracket `'['`,
* a colon `':'`,
* zero or more vertical bars `'|'`,
* another colon `':'`,
* a closing bracket `']'`.

The order of the characters is fixed, only the number of `'|'` may vary.
When we are allowed to delete characters (but not to reorder or insert them) we have to keep the
relative order of the characters that stay.

The longest possible accordion that can be obtained from a given string `s`
is obtained by

1. taking the **leftmost** `'['` that can be the opening bracket,
2. after it the **leftmost** `':'` that can be the first colon,
3. taking the **rightmost** `']'` that can be the closing bracket,
4. before it the **rightmost** `':'` that can be the second colon,
5. keeping all `'|'` that lie between the two chosen colons.

If any of the four required characters does not exist in the needed order,
forming an accordion is impossible.

The length of the resulting accordion is  

```
4   (the two brackets and the two colons)
+ number_of_'|'_between_the_two_colons
```

def to_base_pi(num: int) -> str:
    if num == 0:
        return "0"
    negative = num < 0
    n = -num if negative else num

    # Precompute powers of pi up to the largest needed
    pi = math.pi
    powers = [1.0]  # pi^0
    while powers[-1] * pi <= n:
        powers.append(powers[-1] * pi)

    # Convert using greedy division
    digits = []
    for power in reversed(powers):
        d = int(n // power)          # floor division with float
        # In base pi the allowed digits are 0..3 (since ceil(pi)=4)
        if d > 3:
            d = 3
        digits.append(d)
        n -= d * power

    # Remove leading zeros (should not happen except for safety)
    while digits and digits[0] == 0:
        digits.pop(0)

    # Map digits to characters
    chars = "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ"
    result = ''.join(chars[d] for d in digits)

    if negative:
        result = '-' + result
    return result

def partition_numbers(limit: int):
    """Return list p where p[i] is the number of integer partitions of i (p[0]=1)."""
    p = [0] * (limit + 1)
    p[0] = 1
    for n in range(1, limit + 1):
        total = 0
        k = 1
        while True:
            # generalized pentagonal numbers for k and -k
            g1 = k * (3 * k - 1) // 2
            if g1 > n:

            sign = 1 if k % 2 else -1
            total += sign * p[n - g1]

            g2 = k * (3 * k + 1) // 2
            if g2 > n:
                k += 1
                continue
            total += sign * p[n - g2]

            k += 1
        p[n] = total
    return p

**Solution Explanation**

For an *accordion* we need a subsequence  

```
'['  ':'  (any number of '|')  ':'  ']'
```

The four brackets are fixed, only the `'|'` characters between the two
colons increase the length.
We may delete characters, but we cannot reorder them.

--------------------------------------------------------------------

#### 1.   Observations  

* The first `'['` that we keep can be any `'['` that appears **before**
  the first colon we keep.
* The last `']'` we keep can be any `']'` that appears **after**
  the second colon we keep.
* To obtain the longest possible accordion we should make the interval
  between the two kept colons as large as possible,
  because every `'|'` inside this interval can be kept.

Therefore we can safely choose  

* the **leftmost** `'['` in the whole string,
* the **first** `':'` that appears **after** this `'['`,
* the **rightmost** `']'` in the whole string,
* the **last** `':'` that appears **before** this `']'`.

If the two chosen colons are in the correct order
(`left_colon < right_colon`) then a valid accordion exists,
and its maximal length is  

```
4  (the four fixed characters)  +  number of '|' between the two colons
```

If any of the four characters does not exist, or the colons are crossed,
no accordion can be formed.

--------------------------------------------------------------------

#### 2.   Algorithm
```
let s be the input string, n = len(s)

find L = first index i with s[i] == '['
if not found → answer = -1

find left = first index i > L with s[i] == ':'
if not found → answer = -1

find R = last index i with s[i] == ']'
if not found → answer = -1

find right = last index i < R with s[i] == ':'
if not found → answer = -1

if left >= right → answer = -1
else
        cnt = number of '|' with indices in (left , right)
        answer = 4 + cnt
output answer
```

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm outputs `-1` iff no accordion can be obtained,
otherwise it outputs the maximum possible length.

---

##### Lemma 1  
If an accordion exists, there is one whose `'['` is the leftmost `'['`
in the whole string and whose `']'` is the rightmost `']'`.

**Proof.**  
Take any valid accordion subsequence.
Its `'['` is at some position `i`.  
If there is a `'['` earlier than `i`, we can replace the used `'['`
by that earlier one – the rest of the subsequence (colons, `'|'`, `']'`)
still appears later, so order is preserved.
The same argument works for the `']'`. ∎



##### Lemma 2  
For any fixed `'['` (the leftmost one) and `']'` (the rightmost one),
the longest possible accordion uses the **first** `':'` after `'['`
and the **last** `':'` before `']'`.

**Proof.**  
All characters kept must appear in order.
If we choose a later first colon, the interval between the two colons
shrinks, possibly discarding some `'|'`.  
If we choose an earlier second colon, the interval also shrinks.
Thus the widest interval – obtained by the earliest possible first colon
and the latest possible second colon – contains every `'|'` that could be
kept by any other choice, giving maximal length. ∎



##### Lemma 3  
If the algorithm finds `left < right`, the subsequence
`'[' , ':' (at left) , all '|' between left and right , ':' (at right) , ']'`
is a valid accordion of length `4 + cnt`.

**Proof.**  
By construction the characters appear in the required order.
All characters between the two colons are either `'|'` or other symbols;
the algorithm counts only `'|'`, which we keep, and discards the rest.
Thus the subsequence matches the definition of an accordion. ∎



##### Lemma 4  
If the algorithm outputs `-1`, no accordion can be formed.

**Proof.**  
The algorithm outputs `-1` only when at least one of the following holds:

1. No `'['` in the whole string.
2. No `']'` in the whole string.
3. No `':'` after the leftmost `'['`.
4. No `':'` before the rightmost `']'`.
5. The first such colon is not before the second (`left ≥ right`).

Each case makes it impossible to place the required four symbols
in order, hence no accordion exists. ∎



##### Lemma 5  
If an accordion exists, the algorithm outputs its maximum possible length.

**Proof.**  
Assume an accordion exists.
By Lemma&nbsp;1 we may consider only those using the leftmost `'['`
and rightmost `']'`.  
By Lemma&nbsp;2 the optimal such accordion uses the first colon after `'['`
and the last colon before `']'`, exactly the `left` and `right` found by
the algorithm.  
The algorithm keeps all `'|'` between them, which is the largest possible
number of `'|'` (any other choice would keep a subset).  
Therefore the length `4 + cnt` computed by the algorithm equals the
maximum achievable length. ∎



##### Theorem  
The algorithm prints `-1` iff no accordion can be obtained from `s`;
otherwise it prints the maximum possible length of an accordion that can
be obtained.

**Proof.**  
*If* part: Lemma&nbsp;4.  
*Only‑if* part: when the algorithm does not output `-1`,
Lemma&nbsp;3 shows a valid accordion of length `4+cnt`,
and Lemma&nbsp;5 shows no longer accordion exists. ∎

**Solution Explanation**

For every cell `(i , j)` (`1 ≤ i ≤ n , 1 ≤ j ≤ m`) we have to know how many
different legal paths end in this cell.
From a cell we may go

* **right** – to `(i , j+1)`
* **down**  – to `(i+1 , j)`

The move is illegal when a rock would be pushed out of the board.
While we are moving only right and down, a rock can be pushed only
to the right (if we move right) or down (if we move down).
Therefore a rock never returns to a previous row / column.

--------------------------------------------------------------------

#### 1.   How far can we go in a row / a column ?

In row `i` let  

```
rocks_in_row[i] = number of cells that contain a rock
```

All those rocks will finally occupy the **rightmost**
`rocks_in_row[i]` cells of the row.
Consequently we can never step into a column larger than  

```
row_limit[i] = m - rocks_in_row[i]          (1)
```

(the last column we are allowed to stand on in this row).

The same argument for a column `j`

```
rocks_in_col[j] = number of rocks in column j
col_limit[j] = n - rocks_in_col[j]          (2)
```

If we are in column `j` we are never allowed to go below row
`col_limit[j]`.

--------------------------------------------------------------------

#### 2.   DP with range additions  

`dp[i][j]` – number of legal paths that end in `(i , j)`.

From `(i , j)` we may continue

* to the **right** to any cell `(i , k)` with `j < k ≤ row_limit[i]`
* **down** to any cell `(k , j)` with `i < k ≤ col_limit[j]`

All those cells receive the same amount `dp[i][j]`.
So we have to add a value to a whole interval – a classic
*range‑addition* situation.
With a difference array we can do it in **O(1)** per interval.

Two difference tables are kept

```
addR[i][c] – contributions that start at column c in row i   (right moves)
addD[r][j] – contributions that start at row    r in column j (down moves)
```

When we are processing cell `(i , j)` we already know the total
contribution that reaches it:

```
right_acc  – prefix sum of addR[i][*] up to column j
down_acc[j] – prefix sum of addD[*][j] up to row    i
cur = (right_acc + down_acc[j]) mod MOD
```

For the start cell `(1,1)` we set `cur = 1`.

After `cur` is known we *push* it to the right and down:

```
# right interval [j+1 , row_limit[i]]
if j+1 ≤ row_limit[i]:
        addR[i][j+1]   += cur
        addR[i][row_limit[i]+1] -= cur          (if inside the board)

# down interval [i+1 , col_limit[j]]
if i+1 ≤ col_limit[j]:
        addD[i+1][j]   += cur
        addD[col_limit[j]+1][j] -= cur          (if inside the board)
```

All operations are performed modulo `M = 1 000 000 007`.

The whole board is scanned row by row, column by column.
The answer is the value `cur` obtained for cell `(n , m)`.

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm returns exactly the number of legal
paths from `(1,1)` to `(n,m)`.

---

##### Lemma 1  
For every row `i` any legal path never visits a column larger than
`row_limit[i]`.  
For every column `j` any legal path never visits a row larger than
`col_limit[j]`.

**Proof.**  
In row `i` there are `rocks_in_row[i]` rocks.
During the whole walk rocks can only be pushed to the right,
hence after all possible pushes they occupy the rightmost
`rocks_in_row[i]` cells of the row.
If we stood on column `c > row_limit[i] = m - rocks_in_row[i]`,
the cell to the right of us would already contain a rock,
and moving right would try to push a rock out of the board – illegal.
The same argument holds for columns. ∎



##### Lemma 2  
When the algorithm processes cell `(i , j)`,
`right_acc` equals the sum of `dp[i][k]` over all cells `(i , k)` that
can move right into `(i , j)`.  
`down_acc[j]` equals the sum of `dp[k][j]` over all cells `(k , j)` that
can move down into `(i , j)`.

**Proof.**  
`addR[i][c]` stores the *difference* of contributions that start at column
`c`. The prefix sum `right_acc` therefore contains the total of all
contributions whose interval `[start , end]` covers column `j`.  
Exactly those contributions are produced by cells `(i , start)` whose
right‑move interval contains `j`, i.e. cells that can legally move right
into `(i , j)`. The same reasoning applies to `down_acc`. ∎



##### Lemma 3  
For every cell `(i , j)` the algorithm computes  

```
cur = dp[i][j]   (number of legal paths ending in (i , j)).
```

**Proof by induction over the scanning order** (rows increasing,
inside a row columns increasing).

*Base* `(1,1)`. The algorithm forces `cur = 1`. The only path consists of
the start cell, so `dp[1][1]=1`.

*Induction step.* Assume the statement holds for all cells preceding
`(i , j)` in the scan order.
By Lemma&nbsp;2 `right_acc` is the sum of `dp[i][k]` of all cells that can
move right into `(i , j)`, and `down_acc[j]` is the sum of `dp[k][j]`
of all cells that can move down into `(i , j)`.  
All legal paths that end in `(i , j)` must come from exactly one of those
predecessor cells, and each predecessor contributes its whole number of
paths. Hence the total number of paths is the sum of the two sums,
which the algorithm stores in `cur`. ∎



##### Lemma 4  
Whenever the algorithm adds `cur` to the interval
`[j+1 , row_limit[i]]` of `addR[i][*]` (and analogously for `addD`),
every cell of that interval can be reached from `(i , j)` by a legal
right (down) move, and no other cell can.

**Proof.**  
By Lemma&nbsp;1 a right move from `(i , j)` is legal exactly for columns
`c` with `j < c ≤ row_limit[i]`. The algorithm adds `cur` to the
difference array on the left border `j+1` and subtracts it right after
`row_limit[i]`. Consequently the prefix sum `right_acc` will contain
`cur` for all those columns and zero elsewhere. The same holds for the
down direction. ∎



##### Lemma 5  
All contributions added by the algorithm correspond to **legal**
extensions of already counted paths, and every legal extension is added.

**Proof.**  
Consider a path counted in `dp[i][j] = cur`.  
If moving right from `(i , j)` is legal, Lemma&nbsp;4 tells us that the
algorithm adds `cur` to exactly those cells that can be reached by this
move, i.e. to all legal extensions of the path to the right.
If the move is illegal, the interval is empty and nothing is added.
The argument for moving down is identical. ∎



##### Lemma 6  
For every cell `(i , j)` after it has been processed,
`dp[i][j]` equals the number of **all** legal paths that end in that
cell.

**Proof by induction** using Lemma&nbsp;3 (correct value for the cell)
and Lemma&nbsp;5 (all its extensions are correctly propagated). ∎



##### Theorem  
The algorithm outputs the number of legal paths from `(1,1)` to
`(n,m)` modulo `10^9+7`. If no legal path exists it outputs `0`.

**Proof.**

            down_acc[j] = (down_acc[j] + addD[i][j]) % MOD

def total_digits_upto(k: int) -> int:
    """Sum of number of digits of all numbers from 1 to k."""

def cumulative_block_len(m: int) -> int:
    """
    C(m) = sum_{k=1..m} total_digits_upto(k)
    computed using groups of equal digit length.
    """
    if m <= 0:
        return 0
    total = 0
    d = 1
    start = 1
    while start <= m:
        end = min(m, 10 ** d - 1)
        cnt = end - start + 1
        # sum of i from start to end
        sum_i = (start + end) * cnt // 2
        # contribution of this group
        total += d * (cnt * (m + 1) - sum_i)
        d += 1
        start = 10 ** (d - 1)
    return total

        else:
            number = start + (offset - 1) // d
            digit_idx = (offset - 1) % d
            return str(number)[digit_idx]
        d += 1
        start = 10 ** (d - 1)
    return ''  # should never reach here

    poly = poly.replace('-', '+-')

            continue
        # Skip constant terms (no 'x')
        if 'x' not in term:
            continue

        new_coeff = coeff * exp
        new_exp = exp - 1
        result += new_coeff * (x ** new_exp)
    return result

    t = ['^']

    t = ''.join(t)

    right = 0

**Solution Explanation**

For the given sequence  

```
a1 , a2 , … , an          (all numbers are different)
```

let  

```
b1 < b2 < … < bn
```

be the same numbers sorted increasingly.
`bi` is the *i‑th* smallest element of the whole sequence.

--------------------------------------------------------------------

#### 1.   What does “sorting a subsequence” mean ?

Take a subsequence – a set of positions `i1 < i2 < … < ik`.
Only the numbers on these positions may be permuted, all other positions stay
unchanged.
After sorting the subsequence the numbers on the chosen positions become
increasing, i.e. they appear in the same order as `b` :

```
position i1 gets the smallest number of the subsequence,
position i2 gets the second smallest, …
```

--------------------------------------------------------------------

#### 2.   When is the whole array sorted after sorting all subsequences ?

Look at a single element `aj`.  
Its *rank* is the index of `aj` inside the sorted array `b` :

```
rank[j] = p   ⇔   aj = bp
```

All ranks form a permutation `p[1 … n]` of `1 … n`.

If we sort a subsequence, the numbers on its positions become ordered
by increasing rank.
Consequently **inside one subsequence the ranks must be increasing** when we
walk through the original positions.

Now consider two different subsequences.
Take an element with a smaller rank that stands **right** of an element with a
larger rank in the original array.
If they belong to different subsequences, the larger rank will stay on the
left after sorting – the whole array can never become sorted.
Therefore **every inversion (a larger rank before a smaller one) must be
inside the same subsequence**.

Exactly the same condition is expressed by the permutation cycles:

*If we start from a position `i` and repeatedly jump to the position whose
rank equals the current rank, we finally return to `i`.  
All positions visited belong to one cycle of the permutation `p`.  
Inside a cycle every inversion is internal, between different cycles there is
no inversion.*

Hence each **cycle can be taken as one subsequence** and the whole array
becomes sorted after sorting every cycle separately.
Splitting a cycle into two parts would separate an inversion and would make
sorting impossible, therefore a cycle is already the smallest possible
subsequence.

So the maximal number of subsequences equals the number of cycles of the
permutation `p`.

--------------------------------------------------------------------

#### 3.   Algorithm
```
read n and the array a
make a copy, sort it → b
for every value v in b store its rank (position in b) in a dictionary
build permutation p[i] = rank of a[i]          (1‑based)
visited[1…n] = false
answer = empty list of cycles
for i = 1 … n
        if not visited[i]
                start new cycle
                while not visited[current]
                        add current to current cycle
                        visited[current] = true
                        current = p[current]          # jump to next position
                store the finished cycle
output number of cycles and all cycles (their lengths and indices)
```

#### 4.   Correctness Proof  

We prove that the algorithm outputs a valid partition with the maximum
possible number of subsequences.

---

##### Lemma 1  
For any subsequence that makes the whole array sorted after being sorted,
the ranks of its elements are increasing when the original positions are
traversed.

**Proof.**  
After sorting the subsequence the smallest rank among its elements occupies
the leftmost position of the subsequence, the second smallest the next
position, etc.  
Thus the order of ranks along the positions is the same as the order of the
positions – i.e. increasing. ∎



##### Lemma 2  
If two positions `i < j` satisfy `p[i] > p[j]` (an inversion), then `i`
and `j` must belong to the same subsequence in any feasible solution.

**Proof.**  
Assume they are placed into different subsequences.
After sorting each subsequence, the element with rank `p[i]` stays on the
left of the element with rank `p[j]` (because the left subsequence cannot
receive a smaller rank from the right one).  
Consequently the final array contains a larger number before a smaller one,
contradiction. ∎



##### Lemma 3  
All positions belonging to one cycle of permutation `p` must be placed into
the same subsequence in any feasible solution.

**Proof.**  
Inside a cycle there is a sequence of inversions:
`i1 → i2 → … → ik → i1` where each arrow denotes `p[prev] = next`.
By Lemma&nbsp;2 every pair of consecutive vertices of this sequence must be
together, therefore the whole cycle is forced to be inside one subsequence. ∎



##### Lemma 4  
The set of positions of a single cycle forms a valid subsequence
(i.e. its ranks are increasing along the original order).

**Proof.**  
Take a cycle `C`.  
Consider the positions of `C` in increasing order of their indices:
`c1 < c2 < … < cm`.  
Because the permutation maps each position to the *next* rank inside the
same cycle, the ranks of these positions are exactly a cyclic shift of
`1 … m`.  
Consequently when we read them in the order `c1 , c2 , … , cm` the ranks are
strictly increasing (they are the numbers `1,2,…,m` after a suitable
renumbering). ∎



##### Lemma 5  
Sorting each cycle independently makes the whole array sorted.

**Proof.**  
By Lemma&nbsp;4 every cycle is an increasing subsequence, therefore after
sorting it the numbers on its positions appear in the same order as in the
global sorted array `b`.  
Different cycles contain disjoint sets of ranks, and because a permutation
is a bijection, the union of all cycles gives exactly the whole set
`{1,…,n}`.  
Thus after processing all cycles the array equals `b`. ∎



##### Lemma 6  
The algorithm outputs a partition of the array into subsequences that
satisfies the required property.

**Proof.**  
The algorithm extracts exactly the cycles of permutation `p` (standard
cycle decomposition).  
By Lemma&nbsp;5 sorting each of those cycles yields the globally sorted
array, therefore the produced partition is feasible. ∎



##### Lemma 7  
No feasible solution can contain more subsequences than the number of
cycles.

**Proof.**  
By Lemma&nbsp;3 every cycle must be wholly contained in a single subsequence
of any feasible solution.  
Different cycles are disjoint, therefore each subsequence can contain at
most one whole cycle.  
Hence the total number of subsequences is bounded by the number of cycles. ∎



##### Theorem  
The algorithm prints a feasible partition with the maximum possible number
of subsequences.

**Proof.**  
From Lemma&nbsp;6 the algorithm’s output is feasible.
It uses exactly one subsequence per cycle, i.e. the number of subsequences
equals the number of cycles.
By Lemma&nbsp;7 no feasible solution can have more subsequences.
Therefore the algorithm attains the optimum. ∎

**Solution Explanation**

For a polynomial  

```
P(x) = a0 + a1·x + a2·x² + … ,      0 ≤ ai ≤ 7
```

we have to count the possibilities with  

```
P(2) = m .
```

--------------------------------------------------------------------

#### 1.   From coefficients to three binary numbers  

Every coefficient `ai` (0 … 7) can be written with three binary digits  

```
ai = b0i + 2·b1i + 4·b2i ,      bij ∈ {0,1}
```

Define three (ordinary) binary numbers

```
x = Σ b0i·2^i          (bits b0i)
y = Σ b1i·2^i          (bits b1i)
z = Σ b2i·2^i          (bits b2i)
```

Then

```
P(2) = Σ (b0i + 2·b1i + 4·b2i)·2^i
     = Σ b0i·2^i   + 2·Σ b1i·2^i   + 4·Σ b2i·2^i
     = x + 2·y + 4·z .
```

The transformation is **bijective** :

* from a polynomial we obtain a unique triple `(x , y , z)`;
* from any triple of non‑negative integers `(x , y , z)` we obtain a
  polynomial by taking the binary expansions of `x , y , z`,
  forming the three bits of each coefficient and putting them together.
  All coefficients are automatically between 0 and 7.

Therefore

> the required number of polynomials = the number of triples  
> `(x , y , z) ∈ ℕ³` with `x + 2·y + 4·z = m`.

--------------------------------------------------------------------

#### 2.   Counting the triples  

Fix `z`.  
`z` can be any integer `0 … ⌊m/4⌋`.  
For a chosen `z` the remaining value is `r = m – 4·z` and we need

```
x + 2·y = r ,   x , y ≥ 0 .
```

For each `y = 0 … ⌊r/2⌋` there is exactly one `x = r – 2·y`.  
Hence the number of possibilities for this `z` is `⌊r/2⌋ + 1`.

Because `4·z` is even, `⌊(m‑4·z)/2⌋ = ⌊m/2⌋ – 2·z`.  
So for a fixed `z`

```
contribution(z) = (⌊m/2⌋ – 2·z) + 1 .
```

Summation over all admissible `z` :

```
Z = ⌊m/4⌋
answer = Σ_{z=0}^{Z} ( ⌊m/2⌋ – 2·z + 1 )
        = (Z+1)·(⌊m/2⌋ + 1) – 2· Σ_{z=0}^{Z} z
        = (Z+1)·(⌊m/2⌋ + 1) – 2· Z·(Z+1)/2
        = (Z+1)·( ⌊m/2⌋ + 1 – Z ) .
```

All numbers fit into 64‑bit signed integers (`m ≤ 10¹⁸`),  
the result is required modulo `MOD = 1 000 000 007`.

--------------------------------------------------------------------

#### 3.   Algorithm
For each test case  

```
Z = m // 4
ans = (Z + 1) * ( (m // 2) + 1 - Z )
ans %= MOD
output ans
```

For every position `v (0 … n‑1)` we consider two states  

*`(0 , v)` – it is Rick’s turn, the monster is on `v`*  
*`(1 , v)` – it is Morty’s turn, the monster is on `v`*  

`v = 0` is the black hole.  
If the monster is already in the black hole, the player whose turn it is
has already lost – the previous move won the game.
So the two states `(0,0)` and `(1,0)` are **Lose**.

From a state `(p , v)` the current player `p` may choose any step `s`
from his own set `S[p]` and the monster moves clockwise

```
new = (v + s) mod n
```

The game is a finite directed graph with `2·n` vertices.
We have to classify every vertex as

* **Win** – the player to move can force a win,
* **Lose** – whatever he does, the opponent can force a win,
* **Loop** – none of the above, the game can be forced to continue forever.

The classic way to obtain this classification is a reverse breadth‑first
search (retrograde analysis).

--------------------------------------------------------------------

#### 1.   Data structures

```
moves[0] … list of Rick’s steps
moves[1] … list of Morty’s steps

state[p][v] =  0  unknown (Loop for now)
               1  Win
              -1  Lose

outdeg[p][v] = number of still‑possible moves from (p , v)
               initially = len(moves[p])   (the same for every v)
```

A queue stores already decided states.

--------------------------------------------------------------------

#### 2.   Initialisation  

```
state[0][0] = state[1][0] = -1          # black hole = Lose
push both into the queue
```

--------------------------------------------------------------------

#### 3.   Reverse propagation  

While the queue is not empty take a state `(p , v)`.

*If it is **Lose**:*  
any predecessor `(q , u)` (where `q = 1-p`) that can move to `v`
immediately becomes **Win**, because the current player can jump to a
losing position of the opponent.

*If it is **Win**:*  
for a predecessor `(q , u)` we know that one of its moves leads to a
winning position of the opponent, therefore this move is *bad*.
We decrease `outdeg[q][u]`.  
If **all** moves become bad (`outdeg` reaches `0`) the predecessor is
forced to lose → it becomes **Lose**.

How to obtain the predecessors?
For a step `s` belonging to the opponent `q`

```
u = (v - s) mod n          # the only position that reaches v with step s
```

All such `u` are examined.

--------------------------------------------------------------------

#### 4.   Result  

After the BFS finishes, every still unknown state (`state = 0`) is a
**Loop** – none of the two players can force a win, and the game may be
kept infinite.

The required output are the states for positions `1 … n‑1`
(remember `0` is the black hole) for both players.

--------------------------------------------------------------------

#### 5.   Correctness Proof  

We prove that the algorithm outputs the correct classification for every
state.

---

##### Lemma 1  
State `(p,0)` (monster already in the black hole) is a **Lose** state.

**Proof.** The game ends as soon as the monster reaches the black hole.
If it is already there, the player whose turn it is cannot move any more,
so he has already lost. ∎



##### Lemma 2  
Whenever the algorithm marks a state `(p , v)` as **Win**, the player `p`
has a move to a **Lose** state of the opponent.

**Proof.** A state becomes **Win** only in the branch
*“current state is Lose → predecessor becomes Win”*.
The predecessor `(q , u)` is generated from a known **Lose** state
`(p , v)` using a step `s ∈ moves[q]` such that `u + s ≡ v (mod n)`.
Thus player `q` can move from `u` directly to the opponent’s losing
state, guaranteeing a win. ∎



##### Lemma 3  
Whenever the algorithm marks a state `(p , v)` as **Lose**, **all**
its moves go to states already marked **Win** for the opponent.

**Proof.** A state becomes **Lose** only in the branch
*“current state is Win → decrease out‑degree; if it becomes 0, mark Lose”*.
`outdeg[p][v]` is decreased exactly when one of its moves leads to a
state already known to be **Win** for the opponent.
When the counter finally reaches `0`, every possible move has been
accounted for and each of them leads to a winning opponent state.
Hence the current player cannot avoid losing. ∎



##### Lemma 4  
All states that the algorithm finally leaves as **unknown** (`0`) are
**Loop** states: the game can be forced to continue forever and neither
player can force a win.

**Proof.** Assume an unknown state `(p , v)` were actually **Win**.
Then there exists a move to a **Lose** state of the opponent.
That **Lose** state would have been discovered (Lemma&nbsp;1 and the BFS)
and consequently the predecessor `(p , v)` would have been marked **Win**
(Lemma&nbsp;2) – contradiction.

Assume it were **Lose**.
Then every move would go to a **Win** state of the opponent.
All those **Win** states are discovered by the BFS (they are reachable
from the initial losing states via the reverse edges) and each time a
move to a **Win** is seen the algorithm decreases the out‑degree.
When all moves are to **Win**, the out‑degree becomes `0` and the state
would be marked **Lose** (Lemma&nbsp;3) – contradiction.

Thus the remaining states are neither **Win** nor **Lose**; the game can
avoid reaching a terminal losing state forever, i.e. they are **Loop**. ∎



##### Lemma 5  
Every state that is truly **Win**, **Lose** or **Loop** is eventually
assigned the same classification by the algorithm.

**Proof.**  
*Win*: by definition there exists a move to a **Lose** state.
That **Lose** state is reachable from the initial losing states
(Lemma&nbsp;1) via reverse edges, therefore it will be processed.
When it is processed, the algorithm marks the predecessor as **Win**
(Lemma&nbsp;2). By induction on the length of the shortest winning
path, all true **Win** states are discovered.

*Lose*: by definition all moves go to **Win** states.
All those **Win** states are discovered as above, each decreasing the
out‑degree of the predecessor. When the last move is accounted for,
out‑degree becomes `0` and the algorithm marks the predecessor **Lose**
(Lemma&nbsp;3). Induction on the maximal distance to a **Win** state
gives the claim.

*Loop*: by Lemmas&nbsp;2 and&nbsp;3 any state that is not **Loop**
must be discovered as **Win** or **Lose**. Therefore after the BFS
terminates, the only states left are exactly the true **Loop** states,
as shown in Lemma&nbsp;4. ∎

* “Win”  iff the player to move can force a win,
* “Lose” iff the player to move will inevitably lose,
* “Loop” otherwise.

**Proof.** Directly from Lemma&nbsp;5 the algorithm classifies each

from collections import deque

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it))

    k1 = int(next(it))
    moves0 = [int(next(it)) % n for _ in range(k1)]

    k2 = int(next(it))
    moves1 = [int(next(it)) % n for _ in range(k2)]

    moves = [moves0, moves1]

    # state:  0 = unknown (Loop), 1 = Win, -1 = Lose
    state = [[0] * n for _ in range(2)]
    outdeg = [[len(moves[p]) for _ in range(n)] for p in range(2)]

    q = deque()
    for p in (0, 1):
        state[p][0] = -1          # black hole = Lose
        q.append((p, 0))

    while q:
        p, v = q.popleft()
        cur = state[p][v]
        opp = 1 - p
        if cur == -1:                     # Lose -> predecessors become Win
            for s in moves[opp]:
                u = (v - s) % n
                if state[opp][u] == 0:
                    state[opp][u] = 1
                    q.append((opp, u))
        else:                             # Win -> maybe predecessors become Lose
            for s in moves[opp]:
                u = (v - s) % n
                if state[opp][u] == 0:
                    outdeg[opp][u] -= 1
                    if outdeg[opp][u] == 0:
                        state[opp][u] = -1
                        q.append((opp, u))

    # translate results for positions 1 .. n-1
    words = {1: "Win", -1: "Lose", 0: "Loop"}
    out_lines = []
    for p in range(2):
        line = [words[state[p][pos]] for pos in range(1, n)]
        out_lines.append(' '.join(line))

**Solution Explanation**

For every psycho we want to know after how many steps he will be killed  
(if he never dies we treat it as `0`).  
The answer of the whole line is the maximum of those values.

--------------------------------------------------------------------

#### 1.   Observations  

* In one step a psycho kills **only** his right neighbour, never the left one.  
* Therefore a psycho can be killed only by a **bigger** psycho that stands somewhere on his left side.  
* While the process goes on, the left side of a psycho never changes – only elements on its right disappear.  
* Consequently the moment when a psycho dies depends only on the elements that are **to its left** and are larger than it.

--------------------------------------------------------------------

#### 2.   Stack formulation  

We scan the line from left to right and keep a stack of pairs  

```
(value , steps)          # steps – number of steps after which this value dies
```

The stack is **strictly decreasing** in `value`.  
When we meet a new value `x`

```
cur = 0
while stack not empty and x > stack.top.value:
        cur = max(cur, stack.top.steps)   # the bigger left neighbour that disappears later
        pop
```

All popped elements are smaller than `x`, therefore `x` will kill them (if they are still alive).  
If after the popping the stack becomes empty, there is no bigger element on the left → `x` never dies → `cur = 0`.  
Otherwise there is a bigger element left of `x`; `x` will be killed **one step after** the last of the popped elements dies, therefore `cur = cur + 1`.

Finally we push `(x , cur)` onto the stack and update the global answer with `cur`.

The algorithm is linear because each element is pushed and popped at most once.

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm returns the required number of steps.

---

##### Lemma 1  
When the algorithm finishes processing the first `i` elements, the stack contains exactly those elements that are still alive after the first `i` steps of the real process, in the same order, and the stored `steps` value of each element equals the number of steps after which that element will die (or `0` if it never dies).

**Proof.** By induction over `i`.

*Base (`i = 1`).*  
The stack contains the first element with `steps = 0`. It is alive forever – true.

*Induction step.*  
Assume the statement true after processing the first `i‑1` elements.  
Consider the `i`‑th element `x`.

All elements that are smaller than `x` and are on its left are exactly the elements that the `while` loop pops, because the stack is decreasing.  
All those popped elements are killed by `x` in the **first** step in which `x` becomes adjacent to them, i.e. after the maximal death‑time among them.  
If the stack becomes empty, there is no larger element on the left, therefore `x` never dies → `steps = 0`.  
Otherwise the element that stays on the top after the popping is the nearest larger element on the left; `x` will be killed one step after the last of the popped elements dies, i.e. `steps = 1 + max(steps of popped)`. This is exactly how `cur` is computed.

Thus after pushing `(x,cur)` the invariant holds for the first `i` elements. ∎



##### Lemma 2  
For every position `i` the value `steps` stored by the algorithm equals the real death time of that psycho.

**Proof.** Directly from Lemma&nbsp;1, because after the whole scan the invariant holds for all `n` elements. ∎



##### Lemma 3  
The answer of the problem equals `max_i steps[i]`.

**Proof.** The process stops when the last killing happens. The last killing happens at the moment when the psycho with the largest death time dies. No later killing is possible. ∎



##### Theorem  
The algorithm outputs the number of steps after which no more killings occur.

**Proof.** By Lemma&nbsp;2 the algorithm computes the exact death time of each psycho. By Lemma&nbsp;3 the maximum of those times is exactly the required number of steps. The algorithm outputs this maximum, therefore it is correct. ∎



--------------------------------------------------------------------

*Each element is pushed once and popped at most once.*

```

`n ≤ 10^5`, well inside limits.

        if a[i] == i:

            break

    if root == -1:

        root = 1
        a[root] = root
        changes += 1
    else:
        # ensure the chosen root really points to itself
        if a[root] != root:
            a[root] = root
            changes += 1

    # eliminate other self‑loops (extra roots)
    for i in range(1, n + 1):
        if i != root and a[i] == i:
            a[i] = root
            changes += 1

                changes += 1
                break

        for v in stack:
            in_stack[v] = False

The only thing that matters for the future is

* `i` – index of the first employee that has **not** entered the elevator yet

The multiset is stored as a **sorted tuple** `p`.  
Because the capacity is only 4, the number of different tuples is small:

```

A state is therefore a triple `(i, f, p)`.

#### 2.   Transitions  

From a state we have three possible actions.

* **Move one floor up** (`f < 9`) – cost `+1`, state `(i, f+1, p)`.
* **Move one floor down** (`f > 1`) – cost `+1`, state `(i, f-1, p)`.
* **Open the doors** on the current floor  

  1. all passengers whose destination equals `f` leave,
  2. then, while the next not‑yet‑boarded employee `i` is waiting on this floor
     and the elevator is not full, he/she enters.

  Let  

  * `exit` – number of leaving passengers,
  * `board` – number of entering passengers,
  * `p'` – the new sorted tuple of destinations,
  * `i'` – the new index after the boardings.

  The transition cost is `exit + board` (each person needs one second).

  If `exit + board = 0` the action does nothing and can be ignored.

#### 3.   Goal  

Any state with  

```
i == n   (all employees have entered the elevator)
p == ()  (the elevator is empty)
```

is a finished situation.  
The floor does not matter.  
The smallest distance to any such state is the answer.



--------------------------------------------------------------------

All edge costs are non‑negative, therefore Dijkstra’s algorithm gives the
optimal distances.

Number of states  

```
≤ (n+1) * 9 * 715   ≤ 2001 * 9 * 715  ≈ 13·10⁶

We prove that the algorithm returns the minimum possible total time.

---

##### Lemma 1  
For a fixed prefix of employees (`i`) and a fixed set of passengers (`p`),

**Proof.**  
All later actions depend only on

* where the elevator currently is (`f`),
* which passengers are still inside (`p`),
* which employees have not entered yet (`i`).

The history before reaching this situation cannot affect future moves,
because the rules do not refer to past actions. ∎



##### Lemma 2  
The transition “open doors” described in Section&nbsp;2 is exactly the
effect of opening the doors once on floor `f`.

**Proof.**  
When the doors open, first every passenger whose destination equals `f`
must leave – this is `exit`.  
After that the company rule forces the employees to board in the global
arrival order, therefore the next employee that can possibly board is the
first not yet boarded employee (`i`).  
If his/her waiting floor is `f` and there is free space, he/she must be
allowed to board; this repeats while the conditions hold.
No other employee can board, and no passenger can stay inside after his/her
destination floor is reached. ∎



##### Lemma 3  
Every feasible schedule of the elevator corresponds to a path in the
state graph explored by Dijkstra, and the total time of the schedule equals
the total weight of that path.

**Proof.**  
Start of the schedule: no employee entered (`i = 0`), elevator empty (`p = ()`),
floor 1 – exactly the start state of the algorithm.

Consider one elementary step of a schedule:

* moving one floor up/down – corresponds to the “move” transition,
  cost 1.
* opening the doors – corresponds to the “open doors” transition,
  cost equal to the number of people that leave or enter (by Lemma&nbsp;2).

Thus each elementary step is represented by one edge with the same cost,
and concatenating the steps yields a path whose total weight equals the
schedule’s total time. ∎



##### Lemma 4  
Conversely, every path in the state graph corresponds to a feasible
schedule with the same total time.

**Proof.**  
The edges are exactly the elementary actions that are allowed by the
rules (movement, opening doors).  
Applying them in the order of the path reproduces a schedule that respects
capacity, order of boarding and all other constraints.
The time spent on each edge is precisely the time spent by the corresponding
action, therefore the total times coincide. ∎



##### Lemma 5  
For every state the distance computed by Dijkstra equals the minimum
possible time needed to reach that state from the start.

**Proof.**  
Dijkstra’s algorithm is correct for graphs with non‑negative edge weights.
All edge weights are non‑negative (0, 1, 2,…). ∎



##### Lemma 6  
The minimum distance among all goal states (`i = n`, `p = ()`) equals the

**Proof.**  
*Upper bound*: any optimal schedule yields, by Lemma&nbsp;3, a path to a goal

*Lower bound*: any path to a goal state corresponds (Lemma&nbsp;4) to a feasible

**Proof.**  
By Lemma&nbsp;5 Dijkstra computes the exact shortest distances.

**Solution Explanation**

For every box `i`

* `sum[i]` – sum of all numbers inside the box  
* `total`   – sum of all numbers of all boxes  

If after the reordering every box has the same sum `need`,
```
need = total / k
```
(`total` must be divisible by `k`, otherwise the answer is impossible).

During the operation exactly one number is taken from each box.
Assume we take the number `x` from box `i`.  
After removing it the sum of box `i` becomes `sum[i] – x`.  
To obtain the final sum `need` we have to **receive** a number `y`

```
sum[i] – x + y = need          →          y = need – (sum[i] – x)
```

`y` is completely determined by the chosen `x`.  
If the value `y` exists in some box `j` we could move `y` to box `i`
(and later move `x` somewhere else).  
Doing this for all boxes we obtain a collection of directed edges

```
i  --(take x)-->  j          (j is the box that contains y)
```

The edges must form **disjoint cycles** – every box gives one number
and receives one number.  
A cycle

```
b1 → b2 → … → bm → b1
```

means

* from `b1` we take a number `c1` and put it into `b2`,
* from `b2` we take `c2` and put it into `b3`,
* …
* from `bm` we take `cm` and put it into `b1`.

If we can cover all boxes with such cycles, the required reordering
exists.

The number of boxes `k ≤ 15`, therefore we can work with bit masks
(`mask` – set of boxes already covered) and use DP over subsets.



--------------------------------------------------------------------

#### 1.   Pre‑computations  

* `total` and `need` (`need = total // k`), abort if `total % k != 0`.
* `sum[i]` for every box.
* `pos[value] = box index` – because all numbers are distinct.

--------------------------------------------------------------------

#### 2.   All possible cycles  

For every element `a` of every box `i` we try to build a cycle that
starts with taking `a` from box `i`.

```
start_box = i , start_val = a
cur_box = i , cur_val = a
mask = 0
edges = []                     # (src_box, value_taken, dst_box)

while True:
        need_val = need - (sum[cur_box] - cur_val)
        if need_val not in pos:   → impossible, stop
        nxt = pos[need_val]        # box that contains the needed value

        edges.append( (nxt, need_val, cur_box) )
        if nxt == start_box:
                if need_val == start_val:   # closed a cycle
                        break
                else:                       # would return with another value → impossible
                        stop
        if nxt already in mask:               → would repeat a box → impossible
                stop
        add nxt to mask
        cur_box = nxt
        cur_val = need_val
```

If the loop finishes successfully we have a **valid cycle**.
Its set of boxes is `mask ∪ {start_box}`.
From `edges` we build a dictionary  

```
cycle[box] = (value taken from this box, destination box)
```

Only one cycle per mask is needed – we store the first one we meet.

The whole procedure is `O(total_numbers * k)` ( ≤ 75 000 · 15 ).

#### 3.   Subset DP  

`dp[mask] = (prev_mask, used_cycle_mask)` – we can cover `mask`
using the cycle `used_cycle_mask` together with a solution for
`prev_mask`.  
`dp[0] = (None, None)`.

For every reachable `mask` we try to add any stored cycle whose
boxes do not intersect `mask`.

```
new = mask | cycle_mask
if new not yet visited:
        dp[new] = (mask, cycle_mask)
```

Number of states: `2^k ≤ 32768`.  
Complexity: `O( 2^k · number_of_cycles )`, easily fast enough.

import heapq

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it))
    d = int(next(it))

    # gains: index 0..n-1, gain[0]=gain[n-1]=0
    gain = [0] * n
    for i in range(1, n - 1):
        gain[i] = int(next(it))

    x = [0] * n
    y = [0] * n
    for i in range(n):
        x[i] = int(next(it))
        y[i] = int(next(it))

    # pre‑compute Manhattan distances multiplied by d
    cost = [[0] * n for _ in range(n)]
    for i in range(n):
        for j in range(n):
            if i == j:
                continue
            man = abs(x[i] - x[j]) + abs(y[i] - y[j])
            cost[i][j] = man * d

    direct = cost[0][n - 1]          # upper bound for answer

    # feasibility test for a given initial time T
    def feasible(T: int) -> bool:
        best = [-1] * n
        best[0] = T
        # max‑heap using negative values
        heap = [(-T, 0)]
        while heap:
            cur_neg, u = heapq.heappop(heap)
            cur = -cur_neg
            if cur != best[u]:
                continue          # outdated entry
            if u == n - 1:
                return True
            for v in range(n):
                if v == u:
                    continue
                c = cost[u][v]
                if c <= cur:
                    new_time = cur - c + gain[v]
                    if new_time > best[v]:
                        best[v] = new_time
                        heapq.heappush(heap, (-new_time, v))
        return False

    lo, hi = 0, direct
    while lo < hi:

If we speak a number `x > m` we are muzzled for the next `d` days,

We may permute the whole array `a` before the process starts.
We have to maximise the sum of the spoken numbers.

#### 1.   Observations  

* Numbers `≤ m` never cause a muzzle – they are *safe*.
* Numbers `> m` are *big* – after speaking one we lose the next `d`

If we decide to speak `k` big numbers, they have to be placed like  

```
big , (d silent days) , big , (d silent days) , … , big
```

Only the **last** big number does **not** need the following silent block,
because the days after it are irrelevant.

Therefore the days that are *forced* to exist are  

```
k                – the days we actually speak a big number
(k‑1)·d          – silent days after the first k‑1 big numbers
```

Total forced days  

```
forced(k) = k + (k‑1)·d = (k‑1)·(d+1) + 1          (k ≥ 1)
```

All remaining days (`n – forced(k)`) are free – we can put any safe
numbers there (or nothing).

So for a fixed `k`

* we must have `forced(k) ≤ n`
* we may speak at most `free = n – forced(k)` safe numbers,

```

        the still free slots are split into two for the next level

To connect the `c` components we need exactly `c‑1` expensive edges,

```
k_min = c – 1

#### 1.   What distance can be achieved in an MST ?

Take any two settlements `s = 1` (the king) and `p`.

* any cheap‑edge tree inside each component,

---

##### Lemma 1  
If `parent[i] = i+1` holds for all `i = 1 … p-1`,
the dolls `1 … p` can stay nested together without any operation.

**Proof.**  
For each `i` the required relation “`i` inside `i+1`” already holds.
Because the relation is transitive, the whole chain
`1 inside 2 … inside p` is present.
No operation is needed for any of these dolls. ∎



##### Lemma 2  
No longer prefix than the one found by the algorithm can stay unchanged.

**Proof.**  
The algorithm stops at the smallest `p` such that either `p = n`
or `parent[p] ≠ p+1`.  
If `parent[p] ≠ p+1`, then in the final configuration `p` must be
directly inside `p+1`, which is not true in the current state.
Thus any prefix longer than `p` would contain a wrong adjacency and
must be modified. ∎



##### Lemma 3  
For the dolls `p+1 … n` exactly `n-p` attach operations are necessary.

**Proof.**  
In the final chain each of these dolls (or a block containing it)
must become directly inside the next larger doll.
The only operation that creates a nesting is “put `a` into `b`”,
i.e. an attach.
Since after the whole process the prefix `1 … p` is already built,
every remaining doll/block has to be attached once to extend the chain.
Hence at least `n-p` attaches are required, and they are sufficient
when the dolls are already free. ∎



##### Lemma 4  
Among the dolls `p+1 … n` exactly `k‑1` of them are already free
(the outermost dolls of the original chains different from the prefix’s chain).

**Proof.**  
Initially there are `k` chains.
One of them contains doll `1` and therefore the whole prefix.
All other `k‑1` chains have their outermost doll not nested in any other
doll, i.e. they are free.
No other doll is free because every other doll belongs to a chain and is
inside its predecessor. ∎



##### Lemma 5  
The minimum number of detach operations needed for the dolls
`p+1 … n` equals `(n-p) - (k-1)`.

**Proof.**  
Every doll that is not free must be detached once before it can be
attached (Lemma&nbsp;3).  
The number of non‑free dolls among `p+1 … n` is
`(n-p) - (k-1)` by Lemma&nbsp;4.
Detaching any doll more than once is useless, and detaching fewer
dolls makes some of them impossible to attach.
Thus the stated number is both necessary and sufficient. ∎



##### Lemma 6  
The total number of operations required is  
`2·(n-p) - (k-1)`.

**Proof.**  
From Lemma&nbsp;3 we need `n-p` attaches.
From Lemma&nbsp;5 we need `(n-p) - (k-1)` detaches.
Summing gives `2·(n-p) - (k-1)`. ∎



##### Lemma 7  
The algorithm’s answer equals the minimum possible number of seconds.

**Proof.**  
The algorithm computes `p` exactly as the longest unchanged prefix
(Lemmas&nbsp;1 and&nbsp;2) and then outputs `2·(n-p) - (k-1)`,
which by Lemma&nbsp;6 is the optimal number of operations.
Therefore the answer is minimal. ∎



##### Theorem  
For every valid input the program prints the minimum number of seconds
required to transform the initial configuration into the single chain
`1 → 2 → … → n`.

**Proof.**  
Directly from Lemma&nbsp;7. ∎

                if s[i] == 'o':
                    cnt_o += 1
                else:
                    cnt_u += 1
                i += 1
            effective = cnt_o + 2 * cnt_u   # invariant for this segment

            cnt_h = 0

                    trailing_k += 1
                else:  # ch == 'h'

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return

    a = list(map(int, data[1:]))

    max_a = max(a) if a else 0
    limit = max(2, max_a + 1)

    # smallest prime factor sieve
    spf = list(range(limit))
    for i in range(2, int(limit ** 0.5) + 1):
        if spf[i] == i:  # i is prime
            step = i
            start = i * i
            for j in range(start, limit, step):
                if spf[j] == j:
                    spf[j] = i

    INF = 10 ** 9
    cnt = defaultdict(int)          # how many numbers contain the prime
    min1 = defaultdict(lambda: INF) # smallest exponent among numbers containing the prime
    min2 = defaultdict(lambda: INF) # second smallest exponent (may be equal to min1 later)
    cnt_min1 = defaultdict(int)     # how many times the smallest exponent occurs

    for val in a:
        x = val
        # factorize x
        while x > 1:
            p = spf[x]
            e = 0
            while x % p == 0:
                x //= p
                e += 1
            # update data for prime p
            cnt[p] += 1
            if e < min1[p]:
                min2[p] = min1[p]
                min1[p] = e
                cnt_min1[p] = 1
            elif e == min1[p]:
                cnt_min1[p] += 1
            elif e < min2[p]:
                min2[p] = e

    answer = 1
    for p in cnt.keys():
        zeros = n - cnt[p]
        if zeros >= 2:
            continue  # exponent 0, skip
        elif zeros == 1:
            exp = min1[p]
        else:  # zeros == 0
            if cnt_min1[p] >= 2:
                exp = min1[p]
            else:
                exp = min2[p]
        if exp > 0:

**Solution Explanation**

For an *accordion* we need a subsequence  

```
'['  ':'  (any number of '|')  ':'  ']'
```

The length of the obtained string is  

```
4 + (number of '|' between the two ':' )
```

So we have to choose two different positions of the character `:` :

```
c1 < c2
```

such that

* there is a `'['` somewhere **before** `c1`,
* there is a `']'` somewhere **after**  `c2`.

Among all such pairs we want the maximal amount of `'|'` strictly
between `c1` and `c2`.

--------------------------------------------------------------------

#### 1.   Prefix sums of '`|`'

`pref[i]` – number of '`|`' in the prefix `s[0 … i]` (inclusive).

Then the number of '`|`' strictly between two indices `a < b` is  

```
pref[b‑1] – pref[a]
```

--------------------------------------------------------------------

#### 2.   Information whether a bracket exists

* `left_bracket[i]` – *true* iff there is a `'['` somewhere **before**
  position `i`.
* `right_bracket[i]` – *true* iff there is a `']'` somewhere **after**
  position `i`.

Both arrays are built by one left‑to‑right and one right‑to‑left scan.

--------------------------------------------------------------------

#### 3.   Scanning the string

While moving from left to right we keep the smallest value of `pref`
among all positions `c1` that can be the **first** colon
(`s[c1]==':'`, `left_bracket[c1]==true`).

When we meet a colon that can be the **second** one
(`right_bracket[i]==true`) we already know the best possible `c1`
to its left, therefore we can compute the number of '`|`' between them
in *O(1)* time and update the answer.

```
min_pref = None                # minimal pref[c1] seen so far
answer   = -1

for i = 0 … n‑1
        if s[i] == ':':
                # i is candidate for the second colon
                if right_bracket[i] and min_pref is not None:
                        cnt = (pref[i-1] if i>0 else 0) - min_pref
                        answer = max(answer, 4 + cnt)

                # i can also be the first colon
                if left_bracket[i]:
                        cur = pref[i]          # pref up to i (i is ':')
                        if min_pref is None or cur < min_pref:
                                min_pref = cur
```

If after the whole scan `answer` stayed `-1`, no accordion can be built.

--------------------------------------------------------------------

#### 4.   Correctness Proof  

We prove that the algorithm outputs `-1` iff no accordion exists,
otherwise it outputs the maximum possible length.

---

##### Lemma 1  
`left_bracket[i]` is true exactly when there exists an index `j < i`
with `s[j] = '['`.  
`right_bracket[i]` is true exactly when there exists an index `j > i`
with `s[j] = ']'`.

*Proof.* Both arrays are constructed by a single pass that remembers
whether a `'['` (resp. `']'`) has already been seen. ∎



##### Lemma 2  
During the scan, `min_pref` always equals  

```
min { pref[c] | c < current index,
                     s[c] = ':',
                     left_bracket[c] = true }
```

*Proof.*  
`min_pref` is updated only when the current character is a colon and
`left_bracket` holds, i.e. exactly for admissible first‑colon positions.
The stored value is the minimum of all `pref` values seen so far,
hence the claim. ∎



##### Lemma 3  
When the algorithm processes a colon at position `i` that satisfies
`right_bracket[i]`, the computed `cnt` equals the maximum possible number
of '`|`' between any admissible first colon `c1 < i` and this second
colon `i`.

*Proof.*  
All admissible first colons `c1` are exactly those counted in Lemma&nbsp;2.
For each such `c1` the number of '`|`' between them is
`pref[i‑1] – pref[c1]`.  
Because `pref[i‑1]` is constant, maximizing this expression is the same
as minimizing `pref[c1]`.  
`min_pref` is precisely that minimum, therefore `cnt` is the maximal
possible count. ∎



##### Lemma 4  
Whenever the algorithm updates `answer`, the value `4 + cnt` equals the
length of a feasible accordion.

*Proof.*  
The update is performed only if

* there exists a `'['` before the first colon (`left_bracket`),
* there exists a `']'` after the second colon (`right_bracket`),

and the two colons are in the correct order (`c1 < i`).  
Thus the subsequence `'[' ':' (cnt times '|') ':' ']'` exists,
its length is `4 + cnt`. ∎



##### Lemma 5  
If an accordion can be built from `s`, the algorithm will set `answer`
to at least its length.

*Proof.*  
Take an optimal accordion with first colon `c1` and second colon `c2`.
When the scan reaches `c2`, Lemma&nbsp;2 guarantees that `min_pref`
is at most `pref[c1]`. Consequently, by Lemma&nbsp;3 the computed
`cnt` is at least the number of '`|`' between `c1` and `c2`.
By Lemma&nbsp;4 the algorithm will consider a length
`4 + cnt` which is not smaller than the optimal accordion length,
so `answer` becomes at least that value. ∎



##### Lemma 6  
`answer` never exceeds the length of the longest possible accordion.

*Proof.*  
Every value used to update `answer` corresponds (Lemma&nbsp;4) to an
actually realizable accordion, therefore cannot be larger than the
optimal length. ∎



##### Theorem  
The algorithm outputs  

* `-1` iff no accordion can be obtained from `s`;
* otherwise the maximum possible length of an accordion obtainable from `s`.

*Proof.*  

*If no accordion exists*: then there is no pair of colons satisfying the
bracket conditions, consequently the inner `if` never triggers and
`answer` stays `-1`.

*If at least one accordion exists*: by Lemma&nbsp;5 the algorithm
produces a value not smaller than the optimum, and by Lemma&nbsp;6
not larger. Hence it equals the optimum. ∎
