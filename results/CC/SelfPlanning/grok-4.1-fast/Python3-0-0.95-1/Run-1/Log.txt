class SegTree:
    def __init__(self, n):
        self.n = n
        self.inf = n
        self.tree = [0] * (4 * n)
        self.build(1, 0, n - 1)

    def build(self, node, start, end):
        if start == end:
            self.tree[node] = start
            return
        mid = (start + end) // 2
        self.build(2 * node, start, mid)
        self.build(2 * node + 1, mid + 1, end)
        self.tree[node] = min(self.tree[2 * node], self.tree[2 * node + 1])

    def update(self, node, start, end, idx, val):
        if start == end:
            self.tree[node] = val
            return
        mid = (start + end) // 2
        if idx <= mid:
            self.update(2 * node, start, mid, idx, val)
        else:
            self.update(2 * node + 1, mid + 1, end, idx, val)
        self.tree[node] = min(self.tree[2 * node], self.tree[2 * node + 1])

            return self.inf

        p1 = self.query(2 * node, start, mid, l, r)

## Explanation

The reasoning starts by noting that each operation preserves the total sum of tower heights, S = ∑a_i, since blocks are only moved between towers. The ugliness is defined as the difference between the maximum and minimum heights after any number of operations.

To minimize this difference, the heights must be distributed as evenly as possible given the fixed sum S. For n towers, the most even distribution assigns floor(S/n) blocks to some towers and ceil(S/n) to others. Let h = floor(S/n) and r = S % n. Then, r towers get height h+1, and n-r towers get height h.

In this distribution:
- If r == 0, all heights are h, so maximum = minimum = h, and ugliness = 0.
- If r > 0, maximum = h+1 and minimum = h, so ugliness = 1.

This distribution is achievable because operations allow transferring blocks freely between any two towers, and the initial sum S ≥ n (since each a_i ≥ 1) ensures h ≥ 1, avoiding negative heights.

Thus, for each test case:
1. Compute S, the sum of the heights.
2. If S % n == 0, the minimum ugliness is 0.
3. Otherwise, it is 1.

This matches the examples:
- First: S=30, n=3, 30%3=0 → 0.
- Second: S=8, n=4, 8%4=0 → 0.
- Third: S=12, n=5, 12%5=2 → 1.

The constraints form a circular structure. Positions connected by 'E's must share the same value, forming connected components. Each 'N' requires the adjacent positions to have different values.

In a circle:
- If there are 0 'N's, all positions are connected by 'E's into one component, which is consistent (all equal values).
- If there is exactly 1 'N', the rest of the circle consists of 'E's, connecting the two sides of that 'N' into the same component. This forces the adjacent positions to be equal (via the 'E' path) but unequal (via 'N'), creating a contradiction.
- If there are 2 or more 'N's, each 'N' separates distinct components. The complementary arc for any 'N' contains at least one other 'N', preventing a full 'E' path that would merge the sides. Thus, adjacent components across each 'N' are distinct and can be assigned different values (e.g., using distinct integers for each component).
