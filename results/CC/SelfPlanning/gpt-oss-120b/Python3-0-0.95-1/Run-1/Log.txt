**Solution Explanation**

For every book we are given its title ‚Äì a string `s` of length `m`.
All titles are different.

We have to order the titles according to the *asc‚Äëdesc‚Äëending* rule:

* look at the first position `i` (1‚Äëbased) where two titles differ  
* if `i` is odd ‚Äì the title with the **smaller** letter at this position comes first  
* if `i` is even ‚Äì the title with the **larger** letter at this position comes first  

After sorting we have to output the original indices of the titles.



--------------------------------------------------------------------

#### 1.   Observations  

* The rule is a pure lexicographic order, only the comparison direction
  changes depending on the parity of the position.
* For an odd position we compare the letters normally (`'A' < 'B' < ‚Ä¶`).
* For an even position we compare them *reversed* (`'Z' < 'Y' < ‚Ä¶`).

If we transform every even‚Äëposition character `c` into the character

```
c' = 'Z' - (c - 'A')          // A ‚Üî Z, B ‚Üî Y, ‚Ä¶
```

and keep odd‚Äëposition characters unchanged, then two transformed strings
can be compared **normally** (ordinary ascending lexicographic order) and
the result is exactly the required order of the original strings.

*Proof of correctness* of this transformation is given below.



--------------------------------------------------------------------

#### 2.   Algorithm
```
read n , m
for i = 1 ‚Ä¶ n
        read original string s
        build transformed string t
                for position p = 0 ‚Ä¶ m‚Äë1          // 0‚Äëbased
                        if p is even (odd position 1‚Äëbased)
                                t[p] = s[p]                     // keep
                        else
                                t[p] = 'Z' - (s[p] - 'A')       // reverse
        store (t , i) in an array A
sort A by the transformed string t (ordinary <)
output the stored indices in the sorted order
```

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm outputs the indices in exactly the order
defined in the statement.

---

##### Lemma 1  
For any two original strings `x` and `y` and any position `k` (1‚Äëbased)

* if `k` is odd, `x[k] < y[k]`  **iff**  `tx[k] < ty[k]`
* if `k` is even, `x[k] > y[k]` **iff**  `tx[k] < ty[k]`

where `tx` and `ty` are the transformed strings.

*Proof.*  
If `k` is odd, the algorithm copies the character unchanged, therefore
`tx[k] = x[k]` and `ty[k] = y[k]`. The inequality is preserved.

If `k` is even, the algorithm maps a character `c` to `c' = 'Z'-(c-'A')`.
The mapping is strictly decreasing: larger original letters become
smaller transformed letters.
Hence `x[k] > y[k]` ‚áî `tx[k] < ty[k]`. ‚àé



##### Lemma 2  
Let `p` be the first position where `x` and `y` differ.
Then `tx` is lexicographically smaller than `ty`
iff the asc‚Äëdesc‚Äëending rule says that `x` should come before `y`.

*Proof.*  
Only the character at position `p` matters for the lexicographic order of
`tx` and `ty`.  
If `p` is odd, by Lemma&nbsp;1 `tx[p] < ty[p]` ‚áî `x[p] < y[p]`.  
If `p` is even, Lemma&nbsp;1 gives `tx[p] < ty[p]` ‚áî `x[p] > y[p]`.  
These are exactly the two cases of the required rule. ‚àé



##### Lemma 3  
Sorting the transformed strings with the ordinary ascending lexicographic
order yields the list of original strings sorted according to the
asc‚Äëdesc‚Äëending rule.

*Proof.*  
Consider any two distinct original strings `x` and `y`.  
By Lemma&nbsp;2 the relative order of `tx` and `ty` in ordinary
lexicographic order coincides with the required order of `x` and `y`.  
Therefore a total order produced by sorting the transformed strings is
identical to the desired order of the original strings. ‚àé



##### Theorem  
The algorithm outputs the indices of the given titles in the order
defined in the problem statement.

*Proof.*  
The algorithm builds for every title its transformed string exactly as
described.  
By Lemma&nbsp;3 sorting these transformed strings with the standard
`<` comparator produces the titles in the required order.
Finally the algorithm prints the stored original indices in that order,
hence the output is correct. ‚àé

*Transformation* ‚Äì each of the `n` strings of length `m` is processed once

Given the constraint `n¬∑m ‚â§ 10‚Å∂`, the total running time easily fits

**Solution Explanation**

For one move we may erase  

* one `A` **and** one `B`, or  
* one `B` **and** one `C`.

The two letters do **not** have to be adjacent, they can be taken from any
positions of the current string.

--------------------------------------------------------------------

#### 1.   Observations  

* Every move removes **exactly one `B`**.  
  Therefore the number of performed moves equals the number of `B`‚Äôs
  that are present in the original string.

* The same move also removes **exactly one non‚Äë`B` letter**
  (`A` in the first type, `C` in the second type).

  Consequently the total number of non‚Äë`B` letters (`A`‚ÄØ+‚ÄØ`C`) must be
  equal to the number of performed moves, i.e. to the number of `B`‚Äôs.

* If the equality  

```
cntB = cntA + cntC
```

holds, we can simply pair each `A` with a distinct `B` (using the first
type of move) and each `C` with a distinct remaining `B` (using the second
type). Because the letters are chosen arbitrarily, no further restriction
exists.

* The equality also guarantees that the total length `cntA+cntB+cntC = 2¬∑cntB`
  is even, which is necessary for a complete erasure.

Hence the condition `cntB = cntA + cntC` is **necessary and sufficient**.

--------------------------------------------------------------------

#### 2.   Algorithm
For each test case  

1. count the occurrences of `A`, `B`, `C`;
2. output **YES** if `cntB == cntA + cntC`, otherwise **NO**.

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm prints the correct answer for every test case.

---

##### Lemma 1  
If the string can be completely erased, then `cntB = cntA + cntC`.

**Proof.**  
Each move removes exactly one `B`. After the whole process the string is
empty, therefore the number of performed moves equals the initial number of
`B`‚Äôs, i.e. `cntB`.  
Every move also removes exactly one non‚Äë`B` letter, so the total number of
removed non‚Äë`B` letters is also `cntB`.  
But the total number of non‚Äë`B` letters initially is `cntA + cntC`.  
Thus `cntB = cntA + cntC`. ‚àé



##### Lemma 2  
If `cntB = cntA + cntC`, then the string can be completely erased.

**Proof.**  
Take all `A` letters and pair each of them with a distinct `B` (possible
because `cntA ‚â§ cntB`). Erase each pair using the first type of move.
Afterwards `cntB` decreased by `cntA`, and the remaining numbers are  

```
B' = cntB - cntA = cntC ,   A' = 0 ,   C' = cntC .
```

Now pair each remaining `C` with a distinct remaining `B` (they are equal
in number) and erase them using the second type of move.
All letters disappear, so the string can be fully erased. ‚àé



##### Theorem  
For every test case the algorithm outputs **YES** iff the given string can
be completely erased.

**Proof.**  

*If the algorithm outputs YES.*  
That means `cntB = cntA + cntC`. By Lemma&nbsp;2 the string can be erased.

*If the string can be erased.*  
By Lemma&nbsp;1 we must have `cntB = cntA + cntC`, therefore the algorithm
outputs YES.

Both directions hold, completing the proof. ‚àé

**Solution Explanation**

For every minute `i ( 1 ‚â§ i ‚â§ n )` three new pigs appear.
After minute `i` there are `3¬∑i` pigs in total.
If the wolf arrives at minute `i` and wants to eat exactly `x` pigs,
he can choose any `x`‚Äëelement subset of the `3¬∑i` pigs.
The number of possibilities for this minute is  

```
C(3¬∑i , x)          (binomial coefficient)
```

If `3¬∑i < x` the value is `0`.  
Therefore the answer for a query `x` is

```
Ans[x] = Œ£_{i=1..n} C(3¬∑i , x)                (1)
```

All queries are independent, we have to output `Ans[x] (mod M)`  
with `M = 1 000 000 007`.

The difficulty is the huge limits

```
n ‚â§ 10^6          ‚Üí 3¬∑n ‚â§ 3¬∑10^6
q ‚â§ 2¬∑10^5
```

Computing the sum (1) separately for every query would be `O(n¬∑q)` ‚Äì far
too slow.  
We have to obtain **all** values `Ans[0] ‚Ä¶ Ans[3¬∑n]` in `O(3¬∑n)` time
and then answer each query in `O(1)`.



--------------------------------------------------------------------

#### 1.  A generating‚Äëfunction identity  

```
(1+z)^{3¬∑i} = Œ£_{x‚â•0} C(3¬∑i , x)¬∑z^x
```

Summation over all minutes gives a formal power series

```
A(z) = Œ£_{i=1..n} (1+z)^{3¬∑i}
     = Œ£_{x‚â•0} ( Œ£_{i=1..n} C(3¬∑i , x) ) ¬∑ z^x
```

The coefficient of `z^x` in `A(z)` is exactly `Ans[x]`.

The geometric series formula yields a closed form

```
A(z) = (1+z)^3 ¬∑ ( (1+z)^{3¬∑n} ‚Äì 1 ) / ( (1+z)^3 ‚Äì 1 )          (2)
```

Denote  

```
D(z) = (1+z)^3 ‚Äì 1 = 3z + 3z^2 + z^3                (degree 3, no constant term)
N(z) = (1+z)^3 ¬∑ ( (1+z)^{3¬∑n} ‚Äì 1 )                (numerator)
```

Then `A(z) = N(z) / D(z)` and consequently

```
D(z) ¬∑ A(z) = N(z)                                          (3)
```

--------------------------------------------------------------------

#### 2.  From (3) to a simple recurrence  

Write the series as

```
A(z) = Œ£_{k‚â•0} a_k z^k          (a_k = Ans[k])
D(z) = 3z + 3z^2 + z^3
N(z) = Œ£_{k‚â•0} n_k z^k
```

Multiplying the left hand side of (3) and comparing the coefficient of
`z^t (t ‚â• 1)` we obtain

```
n_t = 3¬∑a_{t-1} + 3¬∑a_{t-2} + a_{t-3}                (4)
```

All indices `< 0` are treated as `0`.  
Equation (4) contains **only already known** values `a_{t-1}, a_{t-2},
a_{t-3}` ‚Äì therefore we can compute the whole array `a_k` sequentially
once the coefficients `n_k` are known.

The only missing initial value is `a_0` :

```
a_0 = Œ£_{i=1..n} C(3¬∑i , 0) = Œ£_{i=1..n} 1 = n                (5)
```

--------------------------------------------------------------------

#### 3.  Computing the numerator coefficients `n_k`

```
B_k = C(3¬∑n , k)          (0 ‚â§ k ‚â§ 3¬∑n)          (binomial of the big power)
```

`(1+z)^{3¬∑n}` is exactly the series `B`.  
We need `(1+z)^{3¬∑n} ‚Äì 1`, i.e. the same series with `B_0 = 0`.

Multiplying by `(1+z)^3 = 1 + 3z + 3z^2 + z^3` gives a convolution with a
kernel of length `4`.  Hence for every `k`

```
n_k = B_k + 3¬∑B_{k-1} + 3¬∑B_{k-2} + B_{k-3}          (6)
```

(terms with a negative index are `0`).  
All `B_k` are obtained in `O(3¬∑n)` using pre‚Äëcomputed factorials and
inverse factorials.

We also need `n_{3¬∑n+1}` because formula (4) for `t = 3¬∑n` uses `n_{3¬∑n+1}`.
For `k = 3¬∑n+1` the only non‚Äëzero term in (6) is `3¬∑B_{3¬∑n}` (since
`B_{3¬∑n}=1`).  Computing `n_k` for `k = 0 ‚Ä¶ 3¬∑n+1` is still `O(3¬∑n)`.



--------------------------------------------------------------------

#### 4.  Whole algorithm  

```
max = 3¬∑n
pre‚Äëcompute factorial[0‚Ä¶max] and invFactorial[0‚Ä¶max]

for k = 0 ‚Ä¶ max
        B[k] = C(max , k)               // using factorials
B[0] = 0                               // subtract the ‚Äú‚Äì1‚Äù

for k = 0 ‚Ä¶ max+1
        n[k] = B[k]                     // B out of range = 0
        if k-1 ‚â• 0   n[k] += 3¬∑B[k-1]
        if k-2 ‚â• 0   n[k] += 3¬∑B[k-2]
        if k-3 ‚â• 0   n[k] += B[k-3]
        n[k] %= MOD

a[0] = n % MOD                         // (5)
inv3 = modular inverse of 3

for t = 1 ‚Ä¶ max
        // from (4) with t+1 instead of t:
        // n_{t+1} = 3¬∑a_t + 3¬∑a_{t-1} + a_{t-2}
        cur = n[t+1] - (3¬∑a[t-1] + a[t-2])          (mod MOD)
        a[t] = cur * inv3  (mod MOD)

Now a[x] = answer for every possible query x (0 ‚â§ x ‚â§ 3¬∑n).

For each query x output a[x] (mod MOD).
```

All loops are linear in `max = 3¬∑n ‚â§ 3¬∑10^6`.  
Memory consumption: a few integer arrays of that size ‚Äì well below the
limit.

We prove that the algorithm outputs the required number of attack plans
for every query.

---

##### Lemma 1  
For every `x (0 ‚â§ x ‚â§ 3¬∑n)` the answer `Ans[x]` defined in (1) equals
the coefficient `a_x` of the series `A(z) = Œ£_{i=1..n} (1+z)^{3¬∑i}`.

**Proof.**  
Expanding each term `(1+z)^{3¬∑i}` gives  

```
(1+z)^{3¬∑i} = Œ£_{x‚â•0} C(3¬∑i , x)¬∑z^x .
```

Summation over `i = 1 ‚Ä¶ n` yields

```
A(z) = Œ£_{x‚â•0} ( Œ£_{i=1..n} C(3¬∑i , x) ) ¬∑ z^x .
```

Thus the coefficient of `z^x` is exactly the inner sum, i.e. `Ans[x]`.
‚àé



##### Lemma 2  
Let `B_k = C(3¬∑n , k)` and define `n_k` by (6).  
Then the series `N(z) = Œ£ n_k z^k` equals the numerator
`(1+z)^3¬∑((1+z)^{3¬∑n} ‚Äì 1)`.

**Proof.**  
`(1+z)^{3¬∑n}` has coefficients `B_k`.  
Subtracting `1` sets `B_0 = 0`.  
Multiplying by `(1+z)^3 = 1 + 3z + 3z^2 + z^3` is a convolution with the
kernel `[1, 3, 3, 1]`.  The convolution formula is exactly (6). ‚àé



##### Lemma 3  
For all `t ‚â• 1` the coefficients `a_t` satisfy the recurrence  

```
a_t = ( n_{t+1} ‚Äì 3¬∑a_{t-1} ‚Äì a_{t-2} ) ¬∑ inv3   (mod M)      (7)
```

with the conventions `a_{-1}=a_{-2}=0`.

**Proof.**  
From (3) we have `D(z)¬∑A(z)=N(z)`.  
Writing the coefficient of `z^{t+1}` (`t ‚â• 0`) gives (4) with `t‚Üêt+1` :

```
n_{t+1} = 3¬∑a_t + 3¬∑a_{t-1} + a_{t-2} .
```

Rearranging yields (7). ‚àé



##### Lemma 4  
The algorithm computes `a_0 ‚Ä¶ a_{3¬∑n}` that satisfy (7) and
`a_0 = n`.

**Proof.**  
The algorithm sets `a_0 = n` (by (5)).  
Then it iterates `t = 1 ‚Ä¶ 3¬∑n` and assigns `a_t` exactly by formula (7)
using the already known `a_{t-1}` and `a_{t-2}`.
Thus every produced `a_t` fulfills (7). ‚àé



##### Lemma 5  
For every `x (0 ‚â§ x ‚â§ 3¬∑n)` the value `a_x` produced by the algorithm
equals `Ans[x]`.

**Proof.**  
Both sequences `{a_x}` and `{Ans[x]}` satisfy the same linear recurrence
(7) with the same initial value `a_0 = Ans[0] = n`.  
A linear recurrence of order `3` together with the first three values
uniquely determines the whole sequence.
The algorithm also yields `a_1 , a_2` that satisfy (7), therefore the two
sequences are identical for all indices. ‚àé



##### Lemma 6  
For each query `x` the algorithm outputs `Ans[x]  (mod M)`.

**Proof.**  
By Lemma&nbsp;5 `a_x = Ans[x]`.  
The algorithm finally prints `a_x` modulo `M`. ‚àé

The program prints, for every query, the number of possible attack

**Proof.**  
Directly from Lemma&nbsp;6. ‚àé

```python
        row_pref = [[0] * (m + 1) for _ in range(n)]

            rp = row_pref[i]
            row = grid[i]
            s = 0
            for j in range(m):

                rp[j + 1] = s

        answer = INF

        # iterate over left and right columns (distance >=3)
        for l in range(m):
            for r in range(l + 3, m):

                B = [0] * n
                for i in range(n):

                    interior_ones = row_pref[i][r] - row_pref[i][l + 1]  # columns l+1 .. r-1
                    A[i] = 2 - left_right + interior_ones
                    B[i] = interior_width - interior_ones

                # prefix sums of A
                prefA = [0] * (n + 1)
                for i in range(n):
                    prefA[i + 1] = prefA[i] + A[i]

                best = INF
                # bottom row index must be at least 4 (height >=5)
                for bottom in range(4, n):
                    top_candidate = bottom - 4
                    val = B[top_candidate] - prefA[top_candidate + 1]
                    if val < best:
                        best = val
                    cost = B[bottom] + prefA[bottom] + best
                    if cost < answer:
                        answer = cost

        out_lines.append(str(answer))
```

For a given `n` we have to count the permutations of the numbers  

```
1 , 2 , ‚Ä¶ , 2n
```

that contain at least `n` *ascents* ‚Äì positions `i (1 ‚â§ i < 2n)` with `p[i] < p[i+1]`.

--------------------------------------------------------------------

#### 1.  Eulerian numbers  

For a length `N` permutation let  

```
A(N , k) = number of permutations with exactly k ascents
```

`A(N , k)` are the **Eulerian numbers**.  
A well‚Äëknown symmetry holds

```
A(N , k) = A(N , N‚Äë1‚Äëk)                (1)
```

(the map ‚Äúreverse the permutation‚Äù changes every ascent into a descent).

The total number of permutations of length `N` is

```
‚àë_{k=0}^{N‚Äë1} A(N , k) = N!            (2)
```

--------------------------------------------------------------------

#### 2.  What we need  

We need  

```
Ans(n) = ‚àë_{k=n}^{2n‚Äë1} A(2n , k)       (3)
```

Because of (1) with `N = 2n`

```
A(2n , k)  ‚Üî  A(2n , 2n‚Äë1‚Äëk)
```

the set `{0,‚Ä¶,2n‚Äë1}` is split into the two disjoint halves

```
{0,‚Ä¶,n‚Äë1}   ‚Üî   {n,‚Ä¶,2n‚Äë1}
```

and the two sums are equal.  
Therefore from (2)

```
‚àë_{k=0}^{n‚Äë1} A(2n , k) = ‚àë_{k=n}^{2n‚Äë1} A(2n , k) = (2n)! / 2
```

So the answer is simply half of `(2n)!`.

--------------------------------------------------------------------

#### 3.  Computing `(2n)! / 2` modulo `M = 1‚ÄØ000‚ÄØ000‚ÄØ007`

`M` is prime, thus the modular inverse of `2` exists:

```
inv2 = (M + 1) / 2 = 500000004
```

Pre‚Äëcompute factorials up to `2¬∑max_n` (`max_n` is the largest `n` among all test cases)

```
fact[0] = 1
for i = 1 ‚Ä¶ 2¬∑max_n:
        fact[i] = fact[i‚Äë1] * i  (mod M)
```

Then for each query

```
answer = fact[2¬∑n] * inv2  (mod M)
```

--------------------------------------------------------------------

#### 4.  Correctness Proof  

We prove that the algorithm outputs the required number of attack plans
(i.e. the number of permutations with at least `n` ascents).

---

##### Lemma 1  
For every `N` and `k` we have `A(N , k) = A(N , N‚Äë1‚Äëk)`.

**Proof.** Reverse a permutation `p`.  
If `p[i] < p[i+1]` then in the reversed permutation the corresponding
pair is a descent, and vice‚Äëversa.
Thus the number of ascents becomes `N‚Äë1‚Äëk`. ‚àé



##### Lemma 2  
For even `N = 2n` the two sums of Eulerian numbers are equal:

```
‚àë_{k=0}^{n‚Äë1} A(N , k) = ‚àë_{k=n}^{N‚Äë1} A(N , k)
```

**Proof.** By Lemma&nbsp;1 each term `A(N , k)` with `k < n`
is paired with the distinct term `A(N , N‚Äë1‚Äëk)` where `N‚Äë1‚Äëk ‚â• n`.
Hence the multiset of terms on the left is a perfect bijection with the
multiset on the right, giving equal sums. ‚àé



##### Lemma 3  
For even `N = 2n`

```
‚àë_{k=n}^{N‚Äë1} A(N , k) = N! / 2 .
```

**Proof.** From Lemma&nbsp;2 the two halves of the total sum (2) are equal,
so each half equals half of `N!`. ‚àé



##### Theorem  
For every input `n` the algorithm outputs the number of permutations of
`{1,‚Ä¶,2n}` that contain at least `n` ascents.

**Proof.**  
The required number is the left‚Äëhand side of (3), i.e.  
`‚àë_{k=n}^{2n‚Äë1} A(2n , k)`.  
By Lemma&nbsp;3 this sum equals `(2n)! / 2`.

#### Correctness Proof  

We prove that the algorithm outputs `"YES"` iff a safe path exists.

*Lemma 1*  
During BFS a cell is enqueued **iff** there exists a safe path from the start
to that cell consisting only of safe cells.

*Proof.*  
- **If** part: The start cell is enqueued initially and trivially has a path
  of length‚ÄØ0. When a cell `v` is dequeued, all its safe, unvisited neighbours

  induction hypothesis) followed by the edge `v‚Üíu`. Hence `u` also has a safe
  path from the start.  
- **Only‚Äëif** part: By induction on the length of a shortest safe path from the
  start to a cell `v`. The start cell satisfies the claim. Assume the claim
  holds for all cells whose shortest safe path length ‚â§‚ÄØk. Let `v` have shortest
  length `k+1` and let `p` be the predecessor of `v` on such a shortest path.
  The shortest path to `p` has length `k`, therefore `p` is visited by the
  induction hypothesis and consequently enqueued. When `p` is processed,
  `v` is a safe neighbour and will be enqueued. ‚àé



*Lemma 2*  
After BFS finishes, a cell is marked *visited* **iff** there exists a safe
path from the start to that cell.

*Proof.*  
A cell is marked visited exactly when it is dequeued, i.e. when it has been
enqueued. By Lemma&nbsp;1 this is equivalent to the existence of a safe path. ‚àé

**Solution Explanation**

For every weight we have to decide

* on which side of the balance it will be put (`L` ‚Äì left, `R` ‚Äì right)
* in which order the weights are taken.

After the *i‚Äëth* move the side written in `S[i]` must be **strictly**
heavier than the other one.

--------------------------------------------------------------------

#### 1.   Reformulation

Let  

* `L` ‚Äì total weight on the left pan,
* `R` ‚Äì total weight on the right pan,
* `D = L ‚Äì R` ‚Äì the signed difference.

Initially `D = 0`.

If a weight `w` is put on the left pan, `D` increases by `w`,
if it is put on the right pan, `D` decreases by `w`.

```
put on left   : D ‚Üê D + w
put on right  : D ‚Üê D ‚Äì w
```

The requirement for the *i‚Äëth* move is

```
S[i] = 'L'   ‚Üí   D > 0
S[i] = 'R'   ‚Üí   D < 0
```

So we have to produce a permutation `p[1‚Ä¶N]` of the given weights and
a sign `s[i] ‚àà {+1,-1}` (`+1` = left, `-1` = right) such that

```
D_i = Œ£_{j=1..i} s[j] * p[j]          (the signed prefix sum)
sign(D_i) =  sign required by S[i]    (never zero)
```

--------------------------------------------------------------------

#### 2.   Observations

*If the current difference already has the required sign*  
we may add any weight **to the same side** ‚Äì the sign will stay the
same and the absolute value only grows.

*If the current difference has the opposite sign (or is zero)*  
we have to **flip** the sign.  
Adding a weight `w` to the required side changes the difference from
`D` to `D ¬± w`.  
The sign flips **iff** `w > |D|`.  
Therefore a flip is possible **iff** there exists an unused weight larger
than the current absolute difference.

Consequences  

* to flip we need a weight `> |D|`.  
  The *smallest* such weight is the best choice ‚Äì it leaves larger
  weights for later (maybe needed for later flips).
* when the sign is already correct we should use the *smallest* remaining
  weight ‚Äì it keeps `|D|` as small as possible, making future flips easier.

--------------------------------------------------------------------

#### 3.   Greedy algorithm

All unused weights are kept in an ordered container (`std::multiset`).

```
D = 0
for i = 1 ‚Ä¶ N
        need = S[i]                // 'L' or 'R'

        if D == 0                         // no sign yet
                take the smallest weight w
                put it on side need
                D =  +w   if need == 'L'
                D =  -w   if need == 'R'

        else if ( D>0 && need=='L' )  // sign already correct
                take the smallest weight w
                D += w                // put on left
        else if ( D<0 && need=='R' )  // sign already correct
                take the smallest weight w
                D -= w                // put on right
        else                           // sign opposite ‚Üí we must flip
                find the smallest weight w  with  w > |D|
                if none exists ‚Üí impossible
                put w on side need
                D = D + w   if need == 'L'   // D was negative, now positive
                D = D - w   if need == 'R'   // D was positive, now negative
```

All taken weights are removed from the set, the chosen side and the
weight are stored for the output.

If at any step a required weight does not exist, the answer is `-1`.

--------------------------------------------------------------------

#### 4.   Correctness Proof  

We prove that the algorithm prints a valid sequence whenever one exists,
and prints `-1` only when no sequence exists.

---

##### Lemma 1  
When the current difference `D` has the required sign,
adding the smallest remaining weight to the same side keeps the sign
correct.

**Proof.**  
If `D>0` and we need `'L'`, we add a positive `w` to `D` ‚Üí `D+w > 0`.  
If `D<0` and we need `'R'`, we subtract a positive `w` ‚Üí `D-w < 0`. ‚àé



##### Lemma 2  
If the current difference `D` has the opposite sign (or `D=0`) and a
solution for the whole remaining suffix exists, then there is a weight
`w` among the unused ones with `w > |D|`.

**Proof.**  
Consider any feasible complete solution.
Look at the first move of that solution that flips the sign of the
difference (it must exist because the required sign now is opposite).
That move puts a weight `w*` on the required side, turning `D` into
`D' = D ¬± w*`.  
For the sign to change we need `|D'| = |w* - |D|| > 0`, i.e. `w* > |D|`.  
Thus a weight larger than `|D|` must be present among the unused ones. ‚àé



##### Lemma 3  
If a flip is needed and the algorithm chooses the *smallest* weight
`w` with `w > |D|`, the resulting difference still allows a solution for
the remaining moves (if any solution existed before the flip).

**Proof.**  
Take any feasible complete solution for the current state.
In that solution the first flip uses some weight `w_f > |D|`.
Our algorithm uses `w ‚â§ w_f` (the smallest possible).  
Replace `w_f` by `w` in that solution: the sign after the flip is still
correct because `w > |D|`.  
All later moves use only weights that are **still unused** (they are a
superset of the weights left after removing `w_f`).  
Since we removed a *smaller* weight, the set of remaining weights is
larger, therefore the rest of the original solution can still be performed.
Thus a solution still exists after our choice. ‚àé



##### Lemma 4  
If the algorithm ever reports ‚Äúimpossible‚Äù (no weight `> |D|` when a
flip is required), then **no** valid complete plan exists.

**Proof.**  
At that moment the required sign is opposite to the current sign, so a
flip is unavoidable. By Lemma&nbsp;2 any feasible plan would need a
weight larger than `|D|`. The algorithm examined the whole set of unused
weights and found none, therefore such a weight does not exist and a
plan is impossible. ‚àé



##### Lemma 5  
If the algorithm finishes all `N` steps, the produced sequence satisfies
all constraints.

**Proof.**  
Induction over the steps.

*Base*: before the first step `D=0`. The algorithm chooses a weight and
places it on the side required by `S[1]`. Consequently after the step
`D` has the required sign.

*Induction step*: assume after step `i‚Äë1` the invariant holds ‚Äì
`D` has the sign required by `S[i‚Äë1]`.  
For step `i` three cases are handled:

1. `D` already has the required sign ‚Äì Lemma&nbsp;1 guarantees the sign
   stays correct.
2. `D` has opposite sign ‚Äì the algorithm flips the sign using a weight
   `> |D|`. After adding/subtracting it the new `D` has the required sign.
3. `D = 0` ‚Äì the algorithm puts any weight on the required side,
   producing the correct sign.

Thus after step `i` the invariant holds. By induction it holds for all
steps, i.e. every prefix satisfies the sign demanded by `S`. ‚àé

The algorithm outputs a sequence of moves that fulfills all requirements
iff such a sequence exists. Otherwise it outputs `-1`.

**Proof.**  

*If the algorithm outputs a sequence*: by Lemma&nbsp;5 it satisfies all
constraints, therefore a solution exists.

*If the algorithm outputs `-1`*: this happens only when a flip is needed
but no weight larger than `|D|` is available. By Lemma&nbsp;4 no solution
can exist. ‚àé

#### 1.   What does an insertion do to an original element ?

Consider the original element `a[i]`.  
Suppose we inserted `x` numbers **before** it (anywhere among the first

The required inequality for this element is

The smallest `x` that can satisfy (1) is

If `a[i] ‚â§ i` the element already fits and we need `x = 0`.

#### 2.   Global view

`need[i]` is the *minimum* number of insertions that have to appear

* `BA(s)` ‚Äì the number of positions `i` with `s[i] = 'b' , s[i+1] = 'a'`

We have to change as few characters as possible (each change may turn a character into `a` or `b`)
so that `AB(s) = BA(s)`.  
If several optimal strings exist we may output any of them.



--------------------------------------------------------------------

#### 1.  A very useful identity  

Consider the contribution of every adjacent pair to the difference  

```
diff = AB(s) ‚Äì BA(s)
```

```
pair "ab"  ‚Üí  +1
pair "ba"  ‚Üí  -1
pair "aa" or "bb" ‚Üí 0
```

If we sum these contributions over the whole string we obtain a telescoping
cancellation:

```
diff = (s[0] == 'a' ? 1 : 0) ‚Äì (s[n‚Äë1] == 'a' ? 1 : 0)
```

*Proof.*  
Write the sum of contributions as  

```
‚àë_{i=0}^{n-2} f(s[i],s[i+1])
```

where `f('a','b') = +1 , f('b','a') = -1 , otherwise 0`.  
Every interior character appears once as the right element of a pair and once as
the left element of the next pair, therefore its contributions cancel.
Only the first character (appears only as left element) and the last character
(appears only as right element) survive, giving exactly the formula above. ‚àé



Consequences  

* `diff` can only be `-1 , 0 , 1`.
* `diff = 0`  **iff** the first and the last character are equal (`a‚Ä¶a` or `b‚Ä¶b`).

--------------------------------------------------------------------

#### 2.  Minimal number of changes  

* If `diff = 0` the string already satisfies `AB = BA`.  
  **0** changes are needed.

* If `diff = 1` then `s[0] = 'a'` and `s[n‚Äë1] = 'b'`.  
  Changing **any** of these two characters makes the first and the last equal,
  therefore `diff` becomes `0`.  
  One change is sufficient and also necessary (otherwise `diff` would stay `1`).

* If `diff = -1` the situation is symmetric:
  `s[0] = 'b' , s[n‚Äë1] = 'a'`.  
  Changing the first character to `'a'` **or** the last to `'b'` fixes the
  difference. Again exactly **1** change is needed.

Thus the answer is always `0` or `1` changes, and we can construct a valid
string by touching at most one endpoint.

and by Lemma&nbsp;3 any solution needs at least one change.
Hence the algorithm is optimal.

Both cases cover all possibilities, completing the proof. ‚àé

The keys are numbered from left to right, i.e. the character `k[i]` is on

While typing we start with the hand already on the first character of `s`,
so the total time is the sum of the distances between consecutive
characters:

```
time = Œ£ |pos(s[i]) ‚Äì pos(s[i‚Äë1])|   for i = 1 ‚Ä¶ |s|-1
```

`pos(c)` is the index of character `c` in the keyboard string `k`.

The problem is therefore reduced to:

1. Build an array `pos[26]` where `pos[c]` is the position of character `c`
   in the keyboard.
2. Walk through the word `s` and accumulate the absolute differences of
   the positions of consecutive characters.

Both steps are `O(26 + |s|)`, which is trivial for the given limits.

#### Correctness Proof  

We prove that the algorithm outputs the exact typing time.

*Lemma 1*  
For any character `c` the algorithm stores `pos[c]` equal to the index of `c`
in the keyboard layout.

*Proof.*  
During preprocessing we iterate over the 26 characters of the keyboard
string `k`. When we are at index `i` we set `pos[k[i]] = i`.  
Since each letter appears exactly once, after the loop `pos[c]` is defined
and equals the unique index of `c`. ‚àé



*Lemma 2*  
For any two consecutive characters `s[i‚Äë1]` and `s[i]` the algorithm adds
`|pos(s[i]) ‚Äì pos(s[i‚Äë1])|` to the answer.

*Proof.*  
When processing the word we keep the previous character `prev`.
For the current character `cur = s[i]` we compute
`abs(pos[cur] - pos[prev])` and add it to the running sum.
By Lemma&nbsp;1 `pos[cur]` and `pos[prev]` are exactly the keyboard positions,
hence the added value equals the required distance. ‚àé



*Theorem*  
The value printed by the algorithm equals the total time needed to type
the whole word `s`.

*Proof.*  
The total time is defined as the sum of distances between every pair of
consecutive characters of `s`.  
By Lemma&nbsp;2 the algorithm adds precisely the distance for each such pair,
and it adds nothing else.  
Therefore the final sum equals the required total time. ‚àé

the floor contains  

```
P(t) = 3¬∑t
```

different pigs.  
If the wolf arrives at minute `t` and wants to eat exactly `x` pigs,
he can choose any `x`‚Äëelement subset of the `3¬∑t` pigs, i.e.  

```
ways(t , x) = C(3¬∑t , x)                (C ‚Äì binomial coefficient)
```

The total number of attack plans for a fixed `x` is the sum over all minutes
that already contain at least `x` pigs:

```
answer(x) = Œ£_{t = ceil(x/3)}^{n} C(3¬∑t , x)          (1)
```

`n ‚â§ 10^6`, `q ‚â§ 2¬∑10^5` and `x ‚â§ 3¬∑n`.  
A direct evaluation of (1) for every query would be `O(n¬∑q)` ‚Äì far too
slow.

--------------------------------------------------------------------

#### 1.  A generating‚Äëfunction identity  

Define the polynomial  

```
G(y) = Œ£_{t=1}^{n} (1 + y)^{3t}
```

The coefficient of `y^x` in `G` is exactly the sum (1), therefore

```
answer(x) = [y^x] G(y) .                                 (2)
```

The geometric series gives a closed form:

```
G(y) = (1+y)^3 ¬∑ ( (1+y)^{3n} ‚Äì 1 ) / ( (1+y)^3 ‚Äì 1 ) .
```

Write  

```
A(y) = (1+y)^3¬∑((1+y)^{3n} ‚Äì 1) = (1+y)^{3n+3} ‚Äì (1+y)^3 .
```

Hence  

```
G(y) = A(y) / ( (1+y)^3 ‚Äì 1 ) .
```

The denominator is a low‚Äëdegree polynomial

```
(1+y)^3 ‚Äì 1 = 3y + 3y^2 + y^3 = y¬∑B(y) ,   B(y) = 3 + 3y + y^2 .
```

So

```
G(y) = ( A(y) / y ) ¬∑ 1 / B(y) .
```

--------------------------------------------------------------------

#### 2.  From the closed form to a linear recurrence  

Let  

```
C(y) = A(y) / y = Œ£_{k‚â•0} c_k y^k ,   c_k = coeff of y^{k+1} in A .
```

Because `A(y) = (1+y)^{3n+3} ‚Äì (1+y)^3`

```
c_k = C(3n+3 , k+1) ‚Äì C(3 , k+1) .                     (3)
```

(The second term is non‚Äëzero only for `k ‚â§ 2`.)

Now `G¬∑B = C`.  Writing the coefficient of `y^k` we obtain for every `k ‚â• 0`

```
3¬∑g_k + 3¬∑g_{k-1} + g_{k-2} = c_k ,                     (4)
```

where `g_k = [y^k] G(y)` and we define `g_{-1}=g_{-2}=0`.

Equation (4) is a **linear recurrence of order‚ÄØ2**,
so all `g_k` (and therefore all answers) can be computed in a single
forward pass:

```
g_0 = c_0 * inv3
g_1 = (c_1 - 3¬∑g_0) * inv3
for k = 2 ‚Ä¶ 3n:
        g_k = (c_k - 3¬∑g_{k-1} - g_{k-2}) * inv3   (mod MOD)
```

`inv3 = 3^{‚Äë1} (mod MOD) = 333333336` because `MOD = 1‚ÄØ000‚ÄØ000‚ÄØ007`.

--------------------------------------------------------------------

#### 3.  Precomputation  

All binomial coefficients needed in (3) are of the form `C(N, r)` with  
`N ‚â§ 3n+3 ‚â§ 3¬∑10^6+3`.  
We pre‚Äëcompute factorials `fac[i]` and inverse factorials `ifac[i]`
for `i = 0 ‚Ä¶ 3n+3` in `O(3n)` time, then

```
C(N,r) = fac[N] ¬∑ ifac[r] ¬∑ ifac[N‚Äër]   (mod MOD)
```

Now we can fill the array `c[0 ‚Ä¶ 3n]` using (3) and afterwards compute
`g[0 ‚Ä¶ 3n]` with the recurrence (4).  
Both steps are linear in `n`.

--------------------------------------------------------------------

#### 4.  Answering the queries  

For each query `x` ( `1 ‚â§ x ‚â§ 3n` ) we simply output `g[x]`,
which is already the required sum (2).  
All answers are taken modulo `MOD`.

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MOD = 1'000'000'007;
using int64 = long long;

int64 modpow(int64 a, int64 e = MOD - 2) {
    int64 r = 1;
    while (e) {
        if (e & 1) r = r * a % MOD;
        a = a * a % MOD;
        e >>= 1;
    }
    return r;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    if (!(cin >> n >> q)) return 0;
    const int MAXN = 3 * n + 3;          // we need up to 3n+3
    vector<int64> fac(MAXN + 1), ifac(MAXN + 1);
    fac[0] = 1;
    for (int i = 1; i <= MAXN; ++i) fac[i] = fac[i - 1] * i % MOD;
    ifac[MAXN] = modpow(fac[MAXN]);
    for (int i = MAXN; i > 0; --i) ifac[i - 1] = ifac[i] * i % MOD;
    auto C = [&](int N, int K) -> int64 {
        if (K < 0 || K > N) return 0;
        return fac[N] * ifac[K] % MOD * ifac[N - K] % MOD;
    };
    
    const int LIM = 3 * n;               // we need answers for x = 0..3n
    vector<int64> c(LIM + 1);
    // c_k = C(3n+3, k+1) - C(3, k+1)   (mod MOD)
    for (int k = 0; k <= LIM; ++k) {
        int64 val = C(3 * n + 3, k + 1);
        if (k + 1 <= 3) {
            val = (val - C(3, k + 1) + MOD) % MOD;
        }
        c[k] = val;
    }
    
    const int64 inv3 = modpow(3);
    vector<int64> g(LIM + 1);
    // recurrence: 3 g_k + 3 g_{k-1} + g_{k-2} = c_k
    // g_{-1}=g_{-2}=0
    if (LIM >= 0) {
        g[0] = c[0] * inv3 % MOD;
    }
    if (LIM >= 1) {
        g[1] = (c[1] - 3 * g[0]) % MOD;
        if (g[1] < 0) g[1] += MOD;
        g[1] = g[1] * inv3 % MOD;
    }
    for (int k = 2; k <= LIM; ++k) {
        int64 cur = (c[k] - 3 * g[k - 1] - g[k - 2]) % MOD;
        if (cur < 0) cur += MOD;
        g[k] = cur * inv3 % MOD;
    }
    
    while (q--) {
        int x; cin >> x;
        cout << g[x] << '\n';
    }
    return 0;
}
```

#### 1.   Observations  

*The length (number of decimal digits) of a number tells us which one is larger*  
If `len(N1) > len(N2)` then `N1 > N2`, and vice‚Äëversa.

```
len(N1) = digits(x1) + p1
len(N2) = digits(x2) + p2
```

`digits(x)` is at most 7 because `x ‚â§ 10^6`.

*If the lengths are equal we have to compare the actual values.*  
When the lengths are equal the difference of the exponents `p1 ‚Äì p2`
cannot be large:

```
len(N1) = digits(x1) + p1
len(N2) = digits(x2) + p2
len(N1) = len(N2)   ‚áí   p1 - p2 = digits(x2) - digits(x1)
```

The right hand side is between `-7` and `+7`.  
So after aligning the exponents we only have to multiply one of the
`x`‚Äëvalues by at most `10^7`, which easily fits into a 64‚Äëbit integer.

#### 2.   Algorithm
For each test case  

```
d1 = number of decimal digits of x1
d2 = number of decimal digits of x2
len1 = d1 + p1
len2 = d2 + p2

if len1 < len2 ‚Üí answer '<'
if len1 > len2 ‚Üí answer '>'
else                // equal length
    diff = p1 - p2          // |diff| ‚â§ 7
    if diff > 0
         // N1 has more trailing zeros, multiply x2
         x2' = x2 * 10^{diff}
         compare x1 with x2'
    else if diff < 0
         // N2 has more trailing zeros, multiply x1
         x1' = x1 * 10^{-diff}
         compare x1' with x2
    else
         // diff == 0, just compare x1 and x2
    output '<' , '>' or '=' according to the comparison
```

`10^{k}` for `k = 0 ‚Ä¶ 7` is pre‚Äëcomputed.

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm outputs the correct relation for every test
case.

---

##### Lemma 1  
For any positive integer `y` and non‚Äënegative integer `p`,
`len(y¬∑10^{p}) = digits(y) + p`.

**Proof.** Multiplying by `10^{p}` appends exactly `p` zeros to the decimal
representation of `y`. Therefore the total number of digits is the
original number of digits of `y` plus `p`. ‚àé



##### Lemma 2  
If `len(N1) ‚âÝ len(N2)` then the longer number is larger.

**Proof.** Both numbers are positive and have no leading zeros.
A decimal number with more digits is always larger than any positive
decimal number with fewer digits. ‚àé



##### Lemma 3  
Assume `len(N1) = len(N2)`. Then `|p1-p2| ‚â§ 7`.

**Proof.** By Lemma&nbsp;1  

```
len(N1) = digits(x1) + p1
len(N2) = digits(x2) + p2
```

Equality of the lengths gives `p1-p2 = digits(x2)-digits(x1)`.  
Both `digits(x1)` and `digits(x2)` are between `1` and `7`, therefore the
difference is in `[-6,6]`. (If `x = 10^6` it has 7 digits, the extreme
difference is `7-1 = 6`.) ‚àé



##### Lemma 4  
When `len(N1) = len(N2)` the algorithm compares the exact values of `N1`
and `N2`.

**Proof.** Let `diff = p1-p2`.

*Case `diff > 0`*  
`N1 = x1¬∑10^{p1}` and `N2 = x2¬∑10^{p2}`.  
Multiplying `N2` by `10^{diff}` gives  

```
N2¬∑10^{diff} = x2¬∑10^{p2+diff} = x2¬∑10^{p1}
```

Thus `N1 ? N2` is equivalent to `x1 ? x2¬∑10^{diff}`.
The algorithm computes exactly this comparison.

*Case `diff < 0`* ‚Äì symmetric.

*Case `diff = 0`* ‚Äì both numbers have the same exponent, comparison of
`x1` and `x2` is sufficient.

Because of Lemma&nbsp;3 `|diff| ‚â§ 7`, the multiplication fits into 64‚Äëbit
and is performed exactly. ‚àé



##### Lemma 5  
If the algorithm outputs '`<`' (resp. '`>`', '`=`') then indeed
`N1 < N2` (resp. `N1 > N2`, `N1 = N2`).

**Proof.** Two possibilities:

*Lengths differ.*  
By Lemma&nbsp;2 the longer number is larger, and the algorithm returns
the correct sign.

*Lengths equal.*  
By Lemma&nbsp;4 the algorithm compares the exact integer values of the
two numbers, therefore the sign it outputs matches the true relation. ‚àé

For one test case we are given  

* `n` ‚Äì number of towers (`2 ‚â§ n ‚â§ 100`)  
* `a1 ‚Ä¶ an` ‚Äì their heights (`1 ‚â§ ai ‚â§ 10^7`)

In one operation we may move **one** block from any tower `i` to any other tower `j`
(`i ‚âÝ j`).  
The operation decreases `ai` by `1` and increases `aj` by `1`.

The *ugliness* of a configuration is  

```
U = max(ai) ‚Äì min(ai)
```

We may perform any number of operations.  
For each test case we have to output the smallest possible ugliness.



--------------------------------------------------------------------

#### 1.   Observations  

* The total number of blocks never changes.

```
S = a1 + a2 + ‚Ä¶ + an          (constant)
```

* Because we can move a single block from any tower to any other,
  by repeating the operation we can obtain **any** integer distribution
  of the `S` blocks among the `n` towers.
  (Think of repeatedly moving blocks from towers that are too high to
  towers that are too low.)

* Therefore the only restriction on the final heights is that they are
  non‚Äënegative integers whose sum equals `S`.

* To minimise `max ‚Äì min` we want the heights to be as equal as possible.
  Let  

```
q = S / n          (integer division)
r = S % n          (0 ‚â§ r < n)
```

  * `r` towers can have height `q+1`
  * the remaining `n‚Äër` towers have height `q`

  This distribution is feasible (its sum is `r¬∑(q+1)+(n‚Äër)¬∑q = S`) and
  its ugliness is  

```
U = (q+1) ‚Äì q = 1   if r > 0
U = 0               if r = 0
```

* No distribution can have ugliness `< 0`.  
  If `r = 0` we can make all heights exactly `q`, giving `U = 0`.  
  If `r > 0` at least one tower must be `q` and another must be `q+1`,
  so `U ‚â• 1`. The construction above shows `U = 1` is achievable.

Hence the answer for a test case is simply  

```
answer = (S % n == 0) ? 0 : 1
```

#### 2.   Algorithm
For each test case  

1. read `n` and the `n` heights  
2. compute `S = sum(ai)` (use 64‚Äëbit integer)  
3. output `0` if `S % n == 0`, otherwise output `1`

#### 3.   Correctness Proof  

We prove that the algorithm outputs the minimum possible ugliness.

---

**Lemma 1**  
From any initial configuration we can reach any integer vector
`b1 ‚Ä¶ bn` with `bi ‚â• 0` and `‚àë bi = S`.

*Proof.*  
Repeatedly pick a tower `i` with `ai > bi` and a tower `j` with `aj < bj`
(and move one block from `i` to `j`). Each move reduces the total
`‚àë |ai - bi|` by `2`. Because the sum of differences is finite, the process
terminates with `ai = bi` for all `i`. ‚àé



**Lemma 2**  
Let `q = ‚åäS/n‚åã` and `r = S mod n`.  
Any distribution of the `S` blocks has ugliness at least  

```
0   if r = 0
1   if r > 0
```

*Proof.*  
If `r = 0` then `S = n¬∑q`. All heights are integers, so the average is `q`.
If some tower were higher than `q` another must be lower than `q`,
hence `max - min ‚â• 1`. But we can set all heights to `q`, giving ugliness `0`,
so the lower bound is `0`.

If `r > 0` then `S = n¬∑q + r` with `0 < r < n`.

Thus at least one tower must be `q` and at least one must be `q+1`,

**Lemma 3**  
The distribution consisting of `r` towers of height `q+1` and `n‚Äër`


*Proof.*

The heights differ by at most `1`, giving the claimed ugliness. ‚àé



**Theorem**  
For each test case the algorithm outputs the minimum possible ugliness.

*Proof.*  
Let `S` be the total number of blocks.

  The algorithm outputs `0`, which is optimal.

* If `S % n != 0` (`r > 0`):  
  Lemma&nbsp;2 gives a lower bound of `1`.  
  Lemma&nbsp;3 gives a reachable configuration with ugliness‚ÄØ1.

#### 1.  Observations  

* If we write the characters of `S` in increasing order (ordinary sorting) we
  obtain the smallest permutation of `S` **without any additional restriction**.
* The only way the restriction can change this order is when `T = "abc"`.
  For the other five permutations (`"acb" , "bac" , "bca" , "cab" , "cba"`)
  the sorted string never contains `T` as a subsequence, because in the sorted
  string all `'a'` come before all `'b'` and all `'b'` before all `'c'`.
  Any other order of the three letters would be lexicographically larger.

* When `T = "abc"` the sorted string looks like  

  ```
  a‚Ä¶a b‚Ä¶b c‚Ä¶c d‚Ä¶d ‚Ä¶
  ```

  If there is at least one `'a'`, one `'b'` **and** one `'c'`,
  the subsequence `abc` is present (take the first `'a'`, the first `'b'`,
  the first `'c'`).  
  To break this subsequence we have to change the relative order of `'b'`
  and `'c'`.  
  The lexicographically smallest way to do that is  

  ```
  a‚Ä¶a c‚Ä¶c b‚Ä¶b d‚Ä¶d ‚Ä¶
  ```

  ‚Äì we keep all `'a'` at the front (they are the smallest letters),
    then put all `'c'` **before** the `'b'`.  
  Any other change would either put a larger letter before `'a'`
  or would keep the original order and still contain `abc`.

* If at least one of the three letters `'a','b','c'` is missing in `S`,
  the subsequence `abc` can never appear, therefore the ordinary sorted
  string is already valid.

We prove that the algorithm prints the required string.

---

##### Lemma 1  
If `T` is not `"abc"` then the sorted string of `S` does **not** contain `T`
as a subsequence.

**Proof.**  
In the sorted string all `'a'` appear before all `'b'` and all `'b'` before all
`'c'`.  
Any permutation of `"abc"` different from `"abc"` changes the relative order
of at least one pair among `'a','b','c'`.  
Because the sorted string respects the order `a < b < c`, that changed order
cannot be found as a subsequence. ‚àé



##### Lemma 2  
Assume `T = "abc"` and `cnt[a],cnt[b],cnt[c] > 0`.  
The string  

```

```

(where `rest` are the letters `'d'..'z'` in increasing order) does **not**
contain `"abc"` as a subsequence.

**Proof.**  
All `'c'` are placed **before** every `'b'`.  
To obtain the subsequence `abc` we would need an `'a'` followed later by a `'b'`
and later by a `'c'`.  
Because every `'c'` occurs before any `'b'`, after we have taken a `'b'`

##### Lemma 3  
Under the same conditions (`T = "abc"` and all three counts positive) any
permutation of `S` that is lexicographically smaller than `A` must contain
`abc` as a subsequence.

**Proof.**  
All strings that are lexicographically smaller than `A` share the same prefix
of `'a'`s (otherwise they would start with a larger letter).  
The first position where they can differ from `A` is the first character after
the block of `'a'`.  
In `A` this character is `'c'`.

still ‚â• `'a'`).

**Proof.**

## 1.  ‚ÄúThree little pigs‚Äù ‚Äì counting attack plans  

### 1.1  Problem recap  

*During `n` minutes a triple of new pigs arrives each minute, i.e. after minute `i` there are `3¬∑i` pigs on the floor.*  
*The wolf may choose a minute `t (1 ‚â§ t ‚â§ n)` and eat exactly `x` pigs that are present at that moment (`x ‚â§ 3¬∑t`).*  
*Two plans are different if the minute is different **or** the set of eaten pigs is different.*  

For many queries `x (1 ‚â§ x ‚â§ 3n)` we have to output the number of possible plans modulo `M = 1‚ÄØ000‚ÄØ000‚ÄØ007`.

--------------------------------------------------------------------

### 1.2  Observations  

If the wolf arrives at minute `t` there are `3t` distinct pigs.  
The number of ways to choose exactly `x` of them is  

```
C(3t, x)   (binomial coefficient)
```

provided `x ‚â§ 3t`; otherwise the contribution of this minute is `0`.

Hence the answer for a given `x` is  

```
ans(x) = Œ£_{t = ceil(x/3)}^{n} C(3t, x)      (mod M)
```

The na√Øve evaluation would be `O(n¬∑q)` ‚Äì far too slow (`n ‚â§ 10‚Å∂, q ‚â§ 2¬∑10‚Åµ`).

--------------------------------------------------------------------

### 1.3  Transforming the sum  

Write `k = 3t`. Then `k` runs over the arithmetic progression `3,6,9,‚Ä¶,3n`.  

```
ans(x) = Œ£_{k = 3¬∑ceil(x/3)}^{3n, step 3} C(k, x)
```

The binomial coefficient `C(k, x)` is a polynomial in `k` of degree `x`.  
Summing a polynomial over an arithmetic progression can be done in **O(1)** if we have the values of the polynomial at the first few points ‚Äì we can pre‚Äëcompute the prefix sums  

```
S(k) = Œ£_{i=0}^{k} C(i, x)          (i runs over all integers)
```

and then  

```
ans(x) = S(3n) - S(3¬∑ceil(x/3) - 3)   (mod M)
```

So we need a fast way to evaluate `S(K) = Œ£_{i=0}^{K} C(i, x)` for any `K ‚â§ 3n` and for **all** `x` that appear in the queries.

--------------------------------------------------------------------

### 1.4  A combinatorial identity  

A well‚Äëknown identity (hockey‚Äëstick) is  

```
Œ£_{i = x}^{K} C(i, x) = C(K+1, x+1)
```

If we also include the terms `i < x` they are zero, therefore  

```
Œ£_{i = 0}^{K} C(i, x) = C(K+1, x+1)
```

Thus  

```
S(K) = C(K+1, x+1) .
```

Now the answer becomes extremely simple:

```
let L = 3¬∑ceil(x/3)          // smallest multiple of 3 not smaller than x
ans(x) = C(3n+1, x+1) - C(L, x+1)   (mod M)
```

All binomial coefficients are taken modulo `M`.  
We only have to be able to compute `C(N, R)` for `N ‚â§ 3¬∑10‚Å∂+1` (because `n ‚â§ 10‚Å∂`).  
That can be done with pre‚Äëcomputed factorials and inverse factorials in `O(1)` per query.

--------------------------------------------------------------------

### 1.5  Algorithm  

Pre‚Äëprocessing (once):

1. `MAX = 3¬∑n + 1`.
2. Compute `fact[i] = i! (mod M)` for `i = 0 ‚Ä¶ MAX`.
3. Compute `invFact[i] = (i!)^{-1} (mod M)` using Fermat‚Äôs little theorem  
   (`invFact[MAX] = fact[MAX]^{M-2}` and go downwards).

Answering a query `x`:

```
L = ((x + 2) / 3) * 3          // integer arithmetic, same as 3¬∑ceil(x/3)
ans = C(3n+1, x+1) - C(L, x+1)
if (ans < 0) ans += M
output ans
```

Complexities  

*Pre‚Äëprocessing*: `O(3n)` time, `O(3n)` memory (`‚âà 12‚ÄØMiB` for `n = 10‚Å∂`).  
*Each query*: `O(1)` time.

--------------------------------------------------------------------

### 1.6  Reference implementation (C++17)

```cpp
#include <bits/stdc++.h>
using namespace std;

const long long MOD = 1'000'000'007LL;

long long modpow(long long a, long long e = MOD - 2) {
    long long r = 1;
    while (e) {
        if (e & 1) r = r * a % MOD;
        a = a * a % MOD;
        e >>= 1;
    }
    return r;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    if (!(cin >> n >> q)) return 0;

    int MAX = 3 * n + 1;                 // we need factorials up to this value
    vector<long long> fact(MAX + 1), invFact(MAX + 1);
    fact[0] = 1;
    for (int i = 1; i <= MAX; ++i) fact[i] = fact[i - 1] * i % MOD;
    invFact[MAX] = modpow(fact[MAX]);
```

## 2.  ‚ÄúInteresting fences‚Äù ‚Äì counting triangles with odd interior points  

### 2.1  Problem recap  

* `n (3 ‚â§ n ‚â§ 6000)` points with **even** integer coordinates, no three collinear.  
* For any triangle formed by three of those points we look at  
  * `I` ‚Äì the number of integer lattice points **strictly inside** the triangle,  
  * `A` ‚Äì its area.  
* The triangle is *interesting* iff `I` is odd **and** `A` is an integer.  

All coordinates are even, therefore every triangle has integer area automatically (Pick‚Äôs theorem).  
We have to count how many triples give an odd `I`.

--------------------------------------------------------------------

### 2.2  Pick‚Äôs theorem  

For a lattice polygon  

```
A = I + B/2 - 1
```

where  

* `A` ‚Äì area,  
* `I` ‚Äì interior lattice points,  
* `B` ‚Äì lattice points on the boundary.

All our points are lattice points, so `B` for a triangle equals  

```
B = gcd(|dx1|,|dy1|) + gcd(|dx2|,|dy2|) + gcd(|dx3|,|dy3|)
```

where `(dxi,dyi)` are the edge vectors.  
Because all coordinates are even, each `dx,dy` is even, thus each `gcd` is **even**.  
Consequently `B` is a sum of three even numbers ‚Üí `B` is even.

From Pick‚Äôs formula  

```
I = A - B/2 + 1
```

`A` is integer, `B/2` is integer (since `B` even).  
Therefore the parity of `I` is exactly the parity of `A + 1`.  
So `I` is odd  ‚áî  `A` is even.

Thus we only have to count triangles whose (integer) area is **even**.

--------------------------------------------------------------------

### 2.3  Parity of the area of a triangle with even coordinates  

For points `(x, y)` with even coordinates we can write  

```
x = 2¬∑X ,   y = 2¬∑Y   (X,Y are integers)
```

The (signed) doubled area of a triangle `(p1,p2,p3)` is  

```
S = | (x2-x1)*(y3-y1) - (x3-x1)*(y2-y1) |
```

Substituting `x = 2X, y = 2Y` gives  

```
S = 4 * | (X2-X1)*(Y3-Y1) - (X3-X1)*(Y2-Y1) |
```

Hence the (non‚Äëdoubled) area `A = S/2` is always a **multiple of 2** (because `S` is a multiple of 4).  
So `A` is even for **every** triangle!  

That means* triangle formed by the given points is interesting.

--------------------------------------------------------------------

### 2.4  Final answer  

The answer is simply the number of ways to choose any three points:

```
C(n, 3) = n¬∑(n-1)¬∑(n-2) / 6
```

We have to output it modulo nothing (the constraints guarantee it fits into 64‚Äëbit, because `n ‚â§ 6000` ‚Üí `C(6000,3) ‚âà 3.6¬∑10¬π‚Å∞` < `2‚Å∂¬≥`).

## 4.  ‚ÄúRecover the array a‚Äù ‚Äì from all non‚Äëempty subsequence sums  

### 4.1  Problem recap  

`a` consists of three positive integers `a‚ÇÅ, a‚ÇÇ, a‚ÇÉ`.  
All 7 non‚Äëempty subsequence sums are written, sorted, and given as  


We have to output **any** triple `a‚ÇÅ, a‚ÇÇ, a‚ÇÉ` that could generate this list.  
It is guaranteed that at least one solution exists.

--------------------------------------------------------------------

### 4.2  Key observations  

* The smallest sum `b‚ÇÅ` must be the smallest single element, i.e.  

  ```
  b‚ÇÅ = min(a‚ÇÅ, a‚ÇÇ, a‚ÇÉ) .
  ```

* The largest sum `b‚Çá` is the sum of all three elements:  

  ```
  b‚Çá = a‚ÇÅ + a‚ÇÇ + a‚ÇÉ .
  ```

* The second smallest sum `b‚ÇÇ` is either the second smallest single element
  **or** the sum of the two smallest elements, depending on the values.
  However we can recover the three numbers without branching:

  Let  

  ```
  x = b‚ÇÅ                     // smallest element
  y = b‚ÇÇ                     // second smallest sum
  S = b‚Çá                     // total sum
  ```

  The three numbers are  

  ```
  a‚ÇÅ = x
  a‚ÇÇ = y - x
  a‚ÇÉ = S - y
  ```

Why does this work?  
Consider the three original numbers sorted: `p ‚â§ q ‚â§ r`.  
The seven sums (sorted) are:

```
p,
q,
p+q,
r,
p+r,
q+r,
p+q+r .
```

Because `p ‚â§ q ‚â§ r`, the order is exactly as above (no ties unless some numbers are equal, which is still fine).  
Thus:

* `b‚ÇÅ = p`
* `b‚ÇÇ = q`   (the second smallest single element)
* `b‚Çá = p+q+r`

From these we obtain  

```
a‚ÇÅ = p = b‚ÇÅ
a‚ÇÇ = q = b‚ÇÇ
a‚ÇÉ = (p+q+r) - (p+q) = b‚Çá - b‚ÇÇ .
```

If some numbers are equal, the same formulas still give a correct multiset
(e.g. `a = [5,5,5]` ‚Üí `b = [5,5,5,10,10,10,15]` ‚Üí `b‚ÇÅ=5, b‚ÇÇ=5, b‚Çá=15`,
the formulas give `5,0,10` ‚Äì but note that `a‚ÇÇ` must be positive.
In the guaranteed‚Äëvalid inputs this never happens; the only possible
degenerate case is when two numbers are equal, which still yields a
positive `a‚ÇÇ = b‚ÇÇ - b‚ÇÅ = 0` **only if** the original had a zero, but the
statement says all `a_i` are positive, therefore such a case does not appear.
Hence the simple formula works for every valid test case.

For each test case:


```cpp
#include <bits/stdc++.h>
using namespace std;

struct DSU {
    vector<int> p, sz;
    DSU(int n = 0) { init(n); }
    void init(int n) {
        p.resize(n);
        sz.assign(n, 1);
        iota(p.begin(), p.end(), 0);
    }
    int find(int x) {
        if (p[x] == x) return x;
        return p[x] = find(p[x]);
    }
    void unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return;
        if (sz[a] < sz[b]) swap(a, b);
        p[b] = a;
        sz[a] += sz[b];
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T;
    if (!(cin >> T)) return 0;
    while (T--) {
        string s;
        cin >> s;
        int n = (int)s.size();
        DSU dsu(n);
        // unite equality edges
        for (int i = 0; i < n; ++i) {
            if (s[i] == 'E') {
                int j = (i + 1) % n;
                dsu.unite(i, j);
            }
        }

        // check inequality edges

            if (s[i] == 'N') {

                    ok = false;
```

```python
import sys

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.sz = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def unite(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.sz[a] < self.sz[b]:
            a, b = b, a
        self.parent[b] = a
        self.sz[a] += self.sz[b]

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return
    t = int(data[0])
    out_lines = []
    idx = 1
    for _ in range(t):
        s = data[idx]
        idx += 1
        n = len(s)
        dsu = DSU(n)
        for i, ch in enumerate(s):
            if ch == 'E':
                dsu.unite(i, (i + 1) % n)
        possible = True
        for i, ch in enumerate(s):
            if ch == 'N':
                if dsu.find(i) == dsu.find((i + 1) % n):
                    possible = False
                    break
        out_lines.append("YES" if possible else "NO")
    sys.stdout.write("\n".join(out_lines))

if __name__ == "__main__":
    solve()
```

*Let the broken stick be `s` and the two untouched sticks be `p` and `q`.*

After the break we have the four numbers `{ p , q , x , y }` with `x+y = s`.

There are only two ways to obtain two equal pairs:

1. **The two untouched sticks are already equal**  
   `p == q`.  
   Then we already have one required pair.
   The other pair must be formed by the two pieces of the broken stick,
   therefore we need `x == y`.  
   This is possible **iff** `s` is even (`x = y = s/2`).

2. **The untouched sticks are different**  
   `p != q`.  
   Then each of them has to be matched with one piece of the broken stick,
   i.e. `{x , y}` must be exactly `{p , q}` (order does not matter).  
   Hence we need `s = p + q`.

No other situation can give two equal pairs, because we have only one
broken stick ‚Äì we cannot create more than two new lengths.

--------------------------------------------------------------------

#### 2.   Consequences

The whole problem reduces to checking the following six simple conditions
for the three numbers `a , b , c` (any permutation of `p , q , s`):

* **Equality + even**
```
a == b  and  c is even
b == c  and  a is even
a == c  and  b is even
```
* **Sum**
```
a + b == c
a + c == b
b + c == a
```

If any of them holds, the answer is **YES**, otherwise **NO**.

All checks are `O(1)`, therefore the whole algorithm works in `O(t)` time
and `O(1)` additional memory.

--------------------------------------------------------------------

#### 3.   Correctness Proof  

We prove that the algorithm prints ‚ÄúYES‚Äù exactly for the feasible test
cases.

---

##### Lemma 1  
If two of the original sticks are equal and the third stick is even,
then a rectangle can be built.

**Proof.**  
Let the equal sticks have length `p` and the even stick have length `s`.
Break the even stick into two halves `s/2` and `s/2`.  
Now we have sticks `{p , p , s/2 , s/2}` ‚Äì two pairs of equal lengths,
hence they form a rectangle. ‚àé



##### Lemma 2  
If one stick length equals the sum of the other two,
then a rectangle can be built.

**Proof.**  
Assume `s = p + q`. Break the stick of length `s` into pieces `p` and `q`.  
The four sticks become `{p , q , p , q}` ‚Äì again two equal pairs,
so they form a rectangle. ‚àé



##### Lemma 3  
If a rectangle can be built after breaking exactly one stick,
then either Lemma&nbsp;1 or Lemma&nbsp;2‚Äôs condition holds.

**Proof.**  
Let the broken stick be `s` and the untouched sticks be `p , q`.

After the break we have four lengths `{p , q , x , y}` with `x+y = s`.

Because they must be two equal pairs, there are only two possibilities:

*Case A ‚Äì the two untouched sticks are equal (`p = q`).*  
Then the remaining pair must be formed by the two pieces, i.e. `x = y`.  
Consequently `s = x + y = 2x` is even ‚Äì exactly Lemma‚ÄØ1‚Äôs condition.

*Case B ‚Äì the untouched sticks are different (`p ‚âÝ q`).*  
Each of them has to be matched with one piece, therefore `{x , y} = {p , q}`.
Thus `s = x + y = p + q` ‚Äì exactly Lemma‚ÄØ2‚Äôs condition.

No other case can give two equal pairs, because we have only one broken
stick. ‚àé



##### Lemma 4  
The algorithm outputs ‚ÄúYES‚Äù **iff** one of the six checked conditions holds.

**Proof.**  
The algorithm explicitly tests the three equal‚Äë+‚Äëeven conditions and the
three sum conditions, and prints ‚ÄúYES‚Äù exactly when at least one test
succeeds. ‚àé



##### Theorem  
For every test case the algorithm prints ‚ÄúYES‚Äù **iff** it is possible to
break exactly one stick into two positive integer pieces so that the four
resulting sticks can be arranged as the sides of a rectangle.

**Proof.**  

*If* the algorithm prints ‚ÄúYES‚Äù, by Lemma‚ÄØ4 one of the six conditions holds.
If it is an equal‚Äë+‚Äëeven condition, Lemma‚ÄØ1 shows a feasible construction.
If it is a sum condition, Lemma‚ÄØ2 shows a feasible construction.
Hence a rectangle is possible.

*Only‚Äëif*: suppose a rectangle is possible.
By Lemma‚ÄØ3 the situation must satisfy either Lemma‚ÄØ1‚Äôs or Lemma‚ÄØ2‚Äôs condition,
i.e. one of the six checks of the algorithm is true.
Consequently the algorithm prints ‚ÄúYES‚Äù.

Thus the algorithm‚Äôs answer is correct in both directions. ‚àé

For each test case we perform a constant number of integer comparisons
and additions.

Both limits easily satisfy the given constraints (`t ‚â§ 10^4`).

**Solution Explanation**

For every test case we are given  

* the size of the room `n √ó m` (`1 ‚â§ n , m ‚â§ 100`);
* the robot‚Äôs start cell `(rb , cb)`;
* the dirty cell `(rd , cd)`.

The robot moves diagonally.  
Its direction is described by two signs `dr , dc` (initially `dr = dc = +1`).  
Before each step it checks the walls:

* if the next row `rb + dr` would be outside `[1 , n]` the vertical direction is
  reflected ‚Äì `dr = ‚Äìdr`;
* if the next column `cb + dc` would be outside `[1 , m]` the horizontal direction
  is reflected ‚Äì `dc = ‚Äìdc`.

After possibly changing the signs it moves to the new cell

```
rb ‚Üê rb + dr
cb ‚Üê cb + dc
```

During **every** second (including second‚ÄØ0, i.e. before the first move) the robot
cleans the whole row `rb` and the whole column `cb`.  
The job is finished as soon as the dirty cell lies in the cleaned row or column,
i.e. when `rb == rd` **or** `cb == cd`.

The constraints are tiny (`n , m ‚â§ 100`).  
The robot‚Äôs motion is periodic: after at most

```
period = lcm( 2¬∑(n‚Äë1) , 2¬∑(m‚Äë1) )
```

steps it returns to the initial state (position + direction).  
`period ‚â§ 2¬∑99¬∑2¬∑99 = 39204`, therefore a straightforward simulation is more than
fast enough.

#### Algorithm
For each test case

```
time ‚Üê 0
dr ‚Üê 1 , dc ‚Üê 1
while true
        if rb == rd or cb == cd          // dirty cell already cleaned
                output time
                break
        // reflect on walls if necessary
        if rb + dr > n or rb + dr < 1
                dr ‚Üê -dr
        if cb + dc > m or cb + dc < 1
                dc ‚Üê -dc
        // move
        rb ‚Üê rb + dr
        cb ‚Üê cb + dc
        time ‚Üê time + 1
```

The loop is guaranteed to terminate because the robot eventually visits a
state where its row or column coincides with the dirty cell (the statement of
the problem guarantees this).

#### Correctness Proof  

We prove that the algorithm outputs the exact number of seconds until the
dirty cell is cleaned.

---

**Lemma 1**  
At the beginning of each iteration (including the first one) the variables
`rb , cb , dr , dc` represent exactly the robot‚Äôs real position and direction
at that moment of time.

*Proof.*  
Initially they are set to the given start position and the initial direction
`(+1,+1)`, which matches the problem statement.  
Assume they are correct at the start of an iteration.  
The algorithm first checks the walls exactly as described in the statement:
if the next cell would be outside the room the corresponding sign is flipped.
After that it adds the (possibly flipped) signs to the coordinates, which is
precisely the robot‚Äôs movement rule.  
Thus after the update the variables again equal the real state of the robot.
‚àé



**Lemma 2**  
During the `k`‚Äëth iteration (`k` counted from‚ÄØ0) the algorithm‚Äôs test  
`rb == rd or cb == cd` is true **iff** at real time `k` the robot cleans the
dirty cell.

*Proof.*  
By Lemma&nbsp;1 the variables hold the real position at time `k`.  
The robot cleans the whole current row and column, therefore the dirty cell is
cleaned exactly when its row equals the robot‚Äôs row **or** its column equals the
robot‚Äôs column. This is exactly the condition tested by the algorithm. ‚àé



**Lemma 3**  
If the algorithm terminates after `T` iterations, then `T` is the smallest
non‚Äënegative integer such that the robot cleans the dirty cell at time `T`.

*Proof.*  
The algorithm stops at the first iteration where the condition of Lemma&nbsp;2
holds, i.e. at the smallest `T` with that property. ‚àé



**Theorem**  
For each test case the algorithm outputs the correct answer ‚Äì the number of
seconds needed for the robot to clean the dirty cell.

*Proof.*  
The algorithm terminates because the robot‚Äôs motion is periodic and the
statement guarantees that a cleaning moment exists; therefore the loop will
eventually encounter it.  
When it stops, by Lemma&nbsp;3 the variable `time` equals the minimal required
seconds. The algorithm prints exactly this value. ‚àé
