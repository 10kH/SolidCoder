import sys

def solve() -> None:
    data = sys.stdin.buffer.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it))
    m = int(next(it))

    items = []
    for idx in range(1, n + 1):
        s = next(it).decode()
        # build transformed string
        # odd positions (1-indexed) -> i even (0-indexed): keep
        # even positions -> i odd: invert alphabet order
        trans_chars = []
        for i, ch in enumerate(s):
            if i & 1:  # even position (1-indexed)
                # map 'A'..'Z' to 'Z'..'A'
                inv = chr(ord('Z') - (ord(ch) - ord('A')))
                trans_chars.append(inv)
            else:
                trans_chars.append(ch)
        trans = ''.join(trans_chars)
        items.append((trans, idx))

    items.sort(key=lambda x: x[0])
    out = ' '.join(str(idx) for _, idx in items)
    sys.stdout.write(out)

if __name__ == "__main__":
    solve()

**Solution Explanation**

For every cell we know whether it already is an obsidian block (`1`) or an empty
block (`0`).  
For a rectangle `a × b` (`a ≥ 5 , b ≥ 4`) we have to pay

* for the **top** and **bottom** rows (without the four corners)  
  `b‑2` cells – they must be `1`
* for the **left** and **right** columns (without the four corners)  
  `a‑2` cells – they must be `1`
* for the whole interior (without the border)  
  `(a‑2)·(b‑2)` cells – they must be `0`

Corners are free, they are never counted.

The task is to obtain the minimum number of cells that have to be flipped
to obtain *any* rectangle with the required shape.



--------------------------------------------------------------------

#### 1.   Prefix sums for rows

`rowPref[i][j]` – number of `1` in row `i` in columns `0 … j‑1`
(`rowPref[i][0] = 0`).  
With it we can obtain in `O(1)`

```
ones in row i, columns L … R   = rowPref[i][R+1] - rowPref[i][L]
```

--------------------------------------------------------------------

#### 2.   Fix the left and right column

Take two columns `L < R` with `R-L+1 ≥ 4` (the width of the rectangle).
All rectangles that use exactly these two columns are examined together.

For every row `i` we pre‑compute two numbers

```
horiz[i]   – cost to make row i the TOP or BOTTOM border
            (all cells between L and R must become 1)

inter[i]   – cost for row i being an interior row
            (cells L and R must become 1, the cells between them must become 0)
```

```
midLen = R-L-1                                 # cells between the two columns
midOnes = rowPref[i][R] - rowPref[i][L+1]      # 1‑s inside the middle part

horiz[i] = (midLen - midOnes)                 # zeros → have to turn to 1

left  = grid[i][L]        # 0 or 1
right = grid[i][R]

inter[i] = (2 - left - right) + midOnes       # left/right to 1  +  middle 1‑s to 0
```

--------------------------------------------------------------------

#### 3.   From rows to a rectangle

For a rectangle whose top row is `t` and bottom row is `b`
(`b - t ≥ 4`, i.e. height `≥ 5`)

```
cost = horiz[t] + horiz[b] + Σ inter[k]   (k = t+1 … b-1)
```

Define a prefix sum of `inter`

```
pref[0] = 0
pref[i+1] = pref[i] + inter[i]          # sum of inter[0 … i]
```

Then  

```
Σ inter[t+1 … b-1] = pref[b] - pref[t+1]
cost = (horiz[t] - pref[t+1]) + (horiz[b] + pref[b])
```

For a fixed bottom row `b` we need the **minimum** value of  
`horiz[t] - pref[t+1]` over all admissible `t ≤ b-4`.
While scanning rows from top to bottom we keep this minimum in a variable
`best`.  
When we move to the next row `b`

```
if b-4 is a valid top row:
        best = min(best, horiz[b-4] - pref[b-3])   # because pref[t+1] = pref[b-3] when t = b-4
answer = min(answer, horiz[b] + pref[b] + best)
```

All operations for the current pair `(L,R)` are `O(n)`.

--------------------------------------------------------------------

#### 4.   Whole algorithm

```
answer = INF
for L = 0 … m-4:
    for R = L+3 … m-1:                # width ≥ 4
        compute horiz[0 … n-1] and inter[0 … n-1]   (O(n))
        build pref[0 … n]                           (O(n))
        best = +INF
        for b = 4 … n-1:                            # bottom row, height ≥5
            t = b-4
            best = min(best, horiz[t] - pref[t+1])
            cur = horiz[b] + pref[b] + best
            answer = min(answer, cur)
print answer
```

in `O(n)`.  
Total operations ≤ `80 000·400 = 32·10⁶`, easily fast enough in Python.

#### 5.   Correctness Proof  

We prove that the algorithm outputs the minimum possible number of flips.

---

##### Lemma 1  
For a fixed pair of columns `(L,R)` and a fixed row `i`

* `horiz[i]` equals the number of cells that must be flipped
  to make row `i` a valid top (or bottom) border of a portal whose
  left/right columns are `L` and `R`.
* `inter[i]` equals the number of cells that must be flipped
  to make row `i` a valid interior row of such a portal.

**Proof.**  
The top/bottom border requires every cell between the two columns
(`midLen = R-L-1` cells) to be `1`.  
`midOnes` of them are already `1`, the remaining `midLen-midOnes` are `0`
and have to be flipped – exactly `horiz[i]`.

For an interior row we need the two border cells `L,R` to be `1`
(`2 - left - right` flips) and all middle cells to be `0`
(`midOnes` flips). Their sum is `inter[i]`. ∎



##### Lemma 2  
For any rectangle with left column `L`, right column `R`,
top row `t` and bottom row `b` (`b-t ≥ 4`) the number of flips needed is  

```
cost = horiz[t] + horiz[b] + Σ_{k=t+1}^{b-1} inter[k]
```

**Proof.**  
By Lemma&nbsp;1 the top row needs `horiz[t]` flips,
the bottom row `horiz[b]` flips,
and each interior row `k` needs `inter[k]` flips.
All required cells are disjoint, therefore the total is the sum. ∎



##### Lemma 3  
For a fixed pair `(L,R)` and a fixed bottom row `b (≥4)` let  

```
best(b) = min_{t ≤ b-4} (horiz[t] - pref[t+1])
```

Then the minimum possible cost of a portal whose right side is column `R`,
left side column `L` and bottom row `b` equals  

```
horiz[b] + pref[b] + best(b)
```

**Proof.**  
Using Lemma&nbsp;2 and the definition of `pref`

```
cost(t,b) = horiz[t] + horiz[b] + pref[b] - pref[t+1]
          = (horiz[t] - pref[t+1]) + (horiz[b] + pref[b])
```

The second term does not depend on `t`.  
Choosing `t` that minimises the first term gives exactly
`best(b)`. ∎



##### Lemma 4  
During the scan of rows for a fixed `(L,R)` the algorithm
maintains `best = best(b)` for the current bottom row `b`.

**Proof.**  
When the loop reaches row `b` we have already processed all rows
`0 … b-1`.  
The candidate top rows for a rectangle ending at `b` are exactly
`0 … b-4`.  
The algorithm updates `best` with the value for `t = b-4`
and never removes any earlier candidate.
Thus after the update `best` equals the minimum over the whole set
`0 … b-4`, i.e. `best(b)`. ∎



##### Lemma 5  
For a fixed pair `(L,R)` the algorithm finds the minimum cost of any
portal that uses exactly these two columns.

**Proof.**  
For every admissible bottom row `b` (loop variable) Lemma&nbsp;4 gives the
correct `best(b)`.  
By Lemma&nbsp;3 the expression `horiz[b] + pref[b] + best` is the minimum
cost of a portal whose bottom side is row `b`.  
The algorithm takes the minimum of this expression over all `b`,
hence obtains the overall minimum for the column pair. ∎



##### Lemma 6  
Among **all** possible portals the algorithm returns the smallest
number of flips.

**Proof.**  
Every portal has some left column `L` and right column `R` with
`R-L+1 ≥ 4`.  
The outer double loop enumerates **all** such pairs, and by Lemma&nbsp;5
for each pair it computes the optimal cost of portals using it.
Taking the minimum over all pairs therefore yields the global optimum. ∎



##### Theorem  
The program prints, for each test case, the minimum possible number of
operations needed to create at least one portal.

**Proof.**  
Directly from Lemma&nbsp;6. ∎

MOD = 10 ** 9 + 7  # not used, kept for compatibility

def solve() -> None:
    it = iter(sys.stdin.read().strip().split())
    t = int(next(it))
    out_lines = []
    for _ in range(t):
        n = int(next(it))
        m = int(next(it))
        grid = [list(map(int, list(next(it).strip()))) for _ in range(n)]

        # row prefix sums: rowPref[i][j] = sum of grid[i][0..j-1]
        rowPref = [[0] * (m + 1) for _ in range(n)]
        for i in range(n):
            rp = rowPref[i]
            row = grid[i]
            s = 0
            for j in range(m):
                s += row[j]
                rp[j + 1] = s

        INF = 10 ** 9
        answer = INF

        # enumerate left and right columns
        for L in range(m - 3):          # need at least width 4
            for R in range(L + 3, m):
                midLen = R - L - 1      # cells between the two columns

                horiz = [0] * n
                inter = [0] * n

                for i in range(n):
                    # ones in the middle part (columns L+1 .. R-1)
                    midOnes = rowPref[i][R] - rowPref[i][L + 1]

                    # cost for being top/bottom border
                    horiz[i] = midLen - midOnes

                    left = grid[i][L]
                    right = grid[i][R]

                    # cost for being interior row
                    inter[i] = (2 - left - right) + midOnes

                # prefix sum of inter
                pref = [0] * (n + 1)
                for i in range(n):
                    pref[i + 1] = pref[i] + inter[i]

                best = INF  # minimal (horiz[t] - pref[t+1]) for admissible t
                # scan bottom row b
                for b in range(4, n):
                    t_idx = b - 4
                    # candidate top row becomes available
                    cand = horiz[t_idx] - pref[t_idx + 1]
                    if cand < best:
                        best = cand

                    cur = horiz[b] + pref[b] + best
                    if cur < answer:
                        answer = cur

        out_lines.append(str(answer))
    sys.stdout.write("\n".join(out_lines))

if __name__ == "__main__":
    solve()

**Solution Explanation**

For every minute `t ( 1 ≤ t ≤ n )` three new pigs appear, therefore after minute `t`
there are `3·t` pigs.
If the wolf arrives at minute `t` and wants to eat exactly `x` pigs,
he can choose any `x` of the `3·t` pigs, i.e.  

```
number of possibilities at minute t = C(3·t , x)
```

The total number of attack plans for a fixed `x` is the sum over all minutes
that already contain at least `x` pigs:

```
answer(x) = Σ_{t = ceil(x/3)}^{n} C(3·t , x)
```

For up to `2·10^5` queries we have to output `answer(x)` for many different
`x ( 1 … 3n )`.  
A direct summation per query is far too slow.



--------------------------------------------------------------------

#### 1.   A generating‑function identity  

Define  

```
F(z) = Σ_{t = 0}^{n} (1+z)^{3t}
```

The coefficient of `z^x` in `F` is exactly `answer(x)` (the term `t = 0`
contributes `0` for `x>0`).  
`F` is a finite geometric series of the polynomial `A(z) = (1+z)^3` :

```
F(z) = ( A(z)^{n+1} – 1 ) / ( A(z) – 1 )
```

Let  

```
B(z) = A(z) – 1 = 3z + 3z² + z³                (no constant term)
G(z) = A(z)^{n+1} – 1 = (1+z)^{3(n+1)} – 1
```

Then  

```
B(z) · F(z) = G(z) .
```

--------------------------------------------------------------------

#### 2.   Coefficient recurrence  

Write the series

```
F(z) = Σ_{k≥0} f_k z^k          (f_k = answer(k) , f_0 = n+1)
G(z) = Σ_{k≥0} g_k z^k          (g_k = C(3(n+1), k) , g_0 = 0)
B(z) = 3z + 3z² + z³
```

Multiplying `B(z)` and `F(z)` and comparing the coefficient of `z^k` :

```
3·f_{k-1} + 3·f_{k-2} + 1·f_{k-3} = g_k          (k ≥ 1)
```

(`f_{negative}` are defined as `0`).  
Therefore for `k ≥ 1`

```
f_k = g_k – 3·f_{k-1} – 3·f_{k-2} – f_{k-3}   (mod MOD)
```

All `g_k` are simple binomial coefficients, they can be obtained in O(1)
with pre‑computed factorials.

We compute `f_0 = n+1` and then iterate `k = 1 … 3n`,
each step needs only a few arithmetic operations – overall **O(n)** time.



--------------------------------------------------------------------

#### 3.   Precomputation  

Maximum needed factorial is `3·(n+1) ≤ 3·10^6 + 3`.  
We pre‑compute

```
fac[i]   = i!          (mod MOD)
ifac[i]  = (i!)^{-1}   (mod MOD)
```

and a helper

```
C(N, K) = fac[N] * ifac[K] % MOD * ifac[N-K] % MOD   (0 ≤ K ≤ N)
```

All arrays have size at most `3·10^6 + 5`, well inside memory limits.



--------------------------------------------------------------------

#### 4.   Answering the queries  

After the single O(n) pass we have the whole array `f[0 … 3n]`.  
For each query `x` we output `f[x]` (mod `1 000 000 007`).

Overall complexity  

```
time   :  O( n + q )          ( n ≤ 10^6 , q ≤ 2·10^5 )
memory :  O( n )   (≈ 3·10^6 integers)
```

Both satisfy the limits.



--------------------------------------------------------------------

#### 5.   Correctness Proof  

We prove that the algorithm outputs the required number of attack plans
for every query.

---

##### Lemma 1  
For every `x (0 ≤ x ≤ 3n)` the coefficient `f_x` of `z^x` in  
`F(z) = Σ_{t=0}^{n} (1+z)^{3t}` equals  

```
f_x = Σ_{t=0}^{n} C(3t , x) .
```

**Proof.**  
The expansion of `(1+z)^{3t}` is `Σ_{k=0}^{3t} C(3t,k) z^k`.  
Summation over `t = 0 … n` collects the coefficient of `z^x` as the sum of
all `C(3t,x)` with `3t ≥ x`. ∎



##### Lemma 2  
For `k ≥ 1` the coefficients satisfy  

```
3·f_{k-1} + 3·f_{k-2} + f_{k-3} = g_k ,
```

where `g_k = C(3(n+1), k)` (and `g_0 = 0`).

**Proof.**  
From the identity `B(z)·F(z) = G(z)` and the explicit forms of the three
polynomials we compare the coefficient of `z^k`.  
`B(z)` contributes `3·f_{k-1}` (term `3z·F`), `3·f_{k-2}` (term `3z²·F`) and
`f_{k-3}` (term `z³·F`).  
The right hand side contributes `g_k`. ∎



##### Lemma 3  
For `k ≥ 1`

```
f_k = g_k – 3·f_{k-1} – 3·f_{k-2} – f_{k-3}   (mod MOD).
```

**Proof.**  
Rearranging the equality of Lemma&nbsp;2 gives the formula. ∎



##### Lemma 4  
The algorithm computes `f_k` for all `k = 0 … 3n` exactly as defined in
Lemma&nbsp;1.

**Proof.**  
*Base*: `f_0` is set to `n+1`. By Lemma&nbsp;1,
`f_0 = Σ_{t=0}^{n} C(3t,0) = Σ_{t=0}^{n} 1 = n+1`.  

*Induction*: Assume the algorithm has already correctly computed
`f_0 … f_{k-1}`.  
It calculates `g_k = C(3(n+1),k)` (the correct coefficient of `G`).  
Then it uses the recurrence of Lemma&nbsp;3, which is mathematically
equivalent to the coefficient identity of Lemma&nbsp;2.
Therefore the obtained `f_k` satisfies Lemma&nbsp;2, and together with the
already correct earlier values it also satisfies Lemma&nbsp;1.
Thus `f_k` is correct. ∎



##### Lemma 5  
For any query `x (1 ≤ x ≤ 3n)` the algorithm outputs  

```
answer(x) = Σ_{t = ceil(x/3)}^{n} C(3t , x) .
```

**Proof.**  
By Lemma&nbsp;1, `f_x = Σ_{t=0}^{n} C(3t , x)`.  
For `t < ceil(x/3)` we have `3t < x` and the binomial coefficient is `0`.  
Hence the sum over all `t = 0 … n` equals the sum over
`t = ceil(x/3) … n`. ∎



##### Theorem  
For every test case and every query the program prints the number of
different attack plans modulo `1 000 000 007`.

**Proof.**  
The program pre‑computes the array `f` exactly as in Lemma&nbsp;4.
For each query `x` it outputs `f[x]`.  
By Lemma&nbsp;5 this value equals the required number of attack plans.
All operations are performed modulo the given prime, therefore the printed
value is the correct answer modulo `10^9+7`. ∎

```python
import sys

MOD = 1_000_000_007

def main() -> None:
    data = sys.stdin.buffer.read().split()
    it = iter(data)
    n = int(next(it))
    q = int(next(it))

    max_k = 3 * n                     # we need answers for 0 .. 3n
    max_fact = 3 * (n + 1)            # factorials up to 3(n+1)

    # ---------- factorials ----------
    fac = [1] * (max_fact + 1)
    for i in range(1, max_fact + 1):
        fac[i] = fac[i - 1] * i % MOD

    ifac = [1] * (max_fact + 1)
    # Fermat inverse for the last factorial
    ifac[max_fact] = pow(fac[max_fact], MOD - 2, MOD)
    for i in range(max_fact, 0, -1):
        ifac[i - 1] = ifac[i] * i % MOD

    def C(N: int, K: int) -> int:
        if K < 0 or K > N:
            return 0
        return fac[N] * ifac[K] % MOD * ifac[N - K] % MOD

    # ---------- g_k = C(3(n+1), k) ----------
    N_big = 3 * (n + 1)
    g = [0] * (max_k + 1)
    for k in range(0, max_k + 1):
        g[k] = C(N_big, k)

    # ---------- compute f_k ----------
    f = [0] * (max_k + 1)
    f[0] = (n + 1) % MOD                     # sum_{t=0}^n C(3t,0)

    for k in range(1, max_k + 1):
        val = g[k]
        if k - 1 >= 0:
            val -= 3 * f[k - 1]
        if k - 2 >= 0:
            val -= 3 * f[k - 2]
        if k - 3 >= 0:
            val -= f[k - 3]
        f[k] = val % MOD

    out_lines = []
    for _ in range(q):
        x = int(next(it))
        out_lines.append(str(f[x]))
    sys.stdout.write("\n".join(out_lines))

if __name__ == "__main__":
    main()

import sys

def sieve(limit: int):
    is_prime = [True] * (limit + 1)
    is_prime[0] = is_prime[1] = False
    for p in range(2, int(limit ** 0.5) + 1):
        if is_prime[p]:
            step = p
            start = p * p
            is_prime[start:limit + 1:step] = [False] * ((limit - start) // step + 1)
    return is_prime

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return

**Solution Explanation**

For every weight we have to decide

* which still unused weight is taken,
* on which side (`L` – left, `R` – right) it is put.

After the `i`‑th move the total weight on the left side minus the total
weight on the right side is called `diff`.

```
put on left   -> diff += w
put on right  -> diff -= w
```

The required condition for the `i`‑th move is

```
S[i] = 'L'   ->   diff > 0
S[i] = 'R'   ->   diff < 0
```

All weights are different and positive.



--------------------------------------------------------------------

#### 1.   Greedy choice

*If the sign of `diff` is already the one we need*  
we may put **any** still unused weight on the same side – the sign will
stay unchanged.
The safest choice is the smallest unused weight.

*If the sign of `diff` is wrong*  
we have to change it.
Assume we need `diff > 0` but currently `diff ≤ 0`.
We put a weight `w` on the left side, afterwards

```
diff' = diff + w   > 0   ⇔   w > -diff
```

So we need the **smallest unused weight larger than `-diff`**.
The same reasoning works for the case `diff < 0`.

If such a weight does not exist the whole task is impossible.



--------------------------------------------------------------------

#### 2.   Data structure – “multiset with “next larger’’ query

All weights are sorted once:

```
weights[0] < weights[1] < … < weights[N-1]
```

During the process some of them are already used.
We need two operations

* `remove(index)` – mark a weight as used,
* `first_unused_ge(pos)` – the smallest unused index `≥ pos`.

Both operations are supported in `O(log N)` by a **Fenwick tree
(Binary Indexed Tree)** that stores for every position `i`
`1` if the weight is still unused, `0` otherwise.

* `prefix_sum(i)` – number of unused weights among the first `i` positions.
* `find_kth(k)` – the index of the `k`‑th unused weight
  (standard binary lifting on the Fenwick tree).

To obtain the smallest unused weight `≥ pos`

```
cnt_before = prefix_sum(pos)          # unused among indices < pos
if total_unused - cnt_before == 0:    # none left
        impossible
k = cnt_before + 1                    # the first unused after pos-1
idx = find_kth(k)                     # 0‑based index of the weight
```

The smallest unused weight at all is simply `find_kth(1)`.



--------------------------------------------------------------------

#### 3.   Algorithm
```
read N, array A, string S
sort A increasingly -> weights
build Fenwick tree with N ones
diff = 0
answer = empty list

for i = 0 … N-1
        if S[i] == 'L'
                if diff > 0                     # sign already correct
                        idx = find_kth(1)       # smallest unused
                        side = 'L'
                        diff += weights[idx]
                else                            # need to become positive
                        need = -diff
                        pos = upper_bound(weights, need)   # first > need
                        idx = first_unused_ge(pos)
                        side = 'L'
                        diff += weights[idx]
        else            # S[i] == 'R'
                if diff < 0
                        idx = find_kth(1)
                        side = 'R'
                        diff -= weights[idx]
                else
                        need = diff
                        pos = upper_bound(weights, need)   # first > need
                        idx = first_unused_ge(pos)
                        side = 'R'
                        diff -= weights[idx]

        record (weights[idx], side) in answer
        Fenwick.add(idx, -1)          # remove this weight
end for

output the recorded pairs
```

If at any step `first_unused_ge` reports “none”, we output `-1`.



--------------------------------------------------------------------

#### 4.   Correctness Proof  

We prove that the algorithm prints a valid sequence
or correctly reports impossibility.

---

##### Lemma 1  
When `diff` already has the required sign,
adding the smallest unused weight on the same side keeps the sign.

**Proof.**  
If the required sign is `>0` then `diff > 0`.  
Putting a positive weight on the left adds a positive number,
so `diff` stays `>0`.  
If the required sign is `<0` the argument is symmetric. ∎



##### Lemma 2  
Assume the required sign is `>0` but currently `diff ≤ 0`.  
Let `w` be the smallest unused weight with `w > -diff`.  
Placing `w` on the left makes the new `diff` positive,
and any other unused weight that also makes `diff` positive is
not smaller than `w`.

**Proof.**  
`diff' = diff + w`. Because `w > -diff`, `diff' > 0`.  
If another weight `w'` also yields a positive `diff'`,
we need `w' > -diff` as well, therefore `w' ≥ w` by minimality of `w`. ∎



##### Lemma 3  
The procedure `first_unused_ge(pos)` returns exactly the smallest
unused weight whose index is at least `pos`, or reports failure
iff no such weight exists.

**Proof.**  
`cnt_before = prefix_sum(pos)` counts unused weights with index `< pos`.  
The `(cnt_before+1)`‑st unused weight is the first one whose index
is `≥ pos`.  
`find_kth(cnt_before+1)` returns its index, because Fenwick’s
`find_kth` is defined to give the position of the *k*-th `1` in the
binary array.  
If `total_unused - cnt_before = 0` there is no unused weight with
index `≥ pos`. ∎



##### Lemma 4  
If the algorithm does not output `-1`, after every processed step `i`
the invariant  

```
diff > 0   iff   S[i] = 'L'
diff < 0   iff   S[i] = 'R'
```

holds.

**Proof by induction over `i`.**

*Base (`i = 0`).*  
The algorithm distinguishes the two cases exactly as described in
Lemma&nbsp;1 or Lemma&nbsp;2 (or the symmetric Lemma for `R`),
therefore after the first move the invariant holds.

*Induction step.*  
Assume the invariant holds after step `i‑1`.  
Consider step `i`.

*Case 1 – required sign already satisfied (`diff` has correct sign).*

The algorithm uses Lemma&nbsp;1, adds the smallest unused weight on the
same side, and the sign stays unchanged – invariant holds.

*Case 2 – required sign not satisfied.*

The algorithm searches the smallest unused weight that is larger than
the absolute value needed (`-diff` for `L`, `diff` for `R`).
By Lemma&nbsp;3 this weight exists **iff** such a weight exists at all.
Placing it on the needed side makes the new `diff` have the required
sign (Lemma&nbsp;2).  
Thus the invariant holds after the move.

If the needed weight does not exist, the algorithm reports `-1`,
which is correct because no move can achieve the required sign
(Lemma&nbsp;2 shows a necessary condition). ∎



##### Lemma 5  
If the algorithm prints `-1`, no valid sequence exists.

**Proof.**  
`-1` is printed only when at some step the algorithm needs a weight
larger than a given threshold (to change the sign) but none is left.
Assume a valid sequence existed.
In that sequence the weight used at this step must also be larger than
the same threshold (otherwise the sign would not change),
contradicting the fact that no such weight remains.
Hence a valid sequence cannot exist. ∎



##### Theorem  
The algorithm outputs a sequence of `N` pairs
`(weight, side)` that satisfies all conditions
iff such a sequence exists; otherwise it outputs `-1`.

**Proof.**  

*If the algorithm outputs a sequence:*  
By Lemma&nbsp;4 the invariant holds after every step,
so the produced sequence fulfills the required inequalities.
All weights are distinct and each is used exactly once
(because we remove it from the Fenwick tree), therefore the output is
valid.

*If the algorithm outputs `-1`:*  
By Lemma&nbsp;5 no valid sequence exists.

Thus the algorithm is correct. ∎

*Sorting* – `O(N log N)`  
*Fenwick tree operations* – each step performs a constant number of
`O(log N)` queries/updates.  
Total time: `O(N log N)`  

Memory consumption: the sorted array (`N` integers) and the Fenwick

import sys
import bisect

MOD = 10 ** 9 + 7

# ---------- Fenwick tree ----------
class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, idx, delta):
        """idx : 0‑based"""
        i = idx + 1
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def sum(self, idx):
        """prefix sum of first idx elements (0‑based, exclusive)"""
        i = idx
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def total(self):
        return self.sum(self.n)

    def find_kth(self, k):
        """return 1‑based index of k‑th 1 (k >= 1)"""
        idx = 0
        bit_mask = 1 << (self.n.bit_length())
        while bit_mask:
            nxt = idx + bit_mask
            if nxt <= self.n and self.bit[nxt] < k:
                k -= self.bit[nxt]
                idx = nxt
            bit_mask >>= 1
        return idx + 1   # 1‑based

*Why?*

*   The sequence that is built must be **strictly increasing**.

*   The largest value of the whole array is the only value that can be taken

*   Consequently the winner is the player that manages to take a

--------------------------------------------------------------------

#### 1.   Which maximum can be taken first ?

Assume the maximum value `mx` occurs at positions  

```
p1 , p2 , … , pk          (0‑based indices)
```

From a position `p` we have to remove all elements on one side of `p`
before `mx` becomes an end of the current segment.
The number of required moves is the distance to the nearer end

```
d(p) = min( p , n‑1‑p )
```

If `d(p)` is **even**, the first player (Alice) can be the one who removes
exactly those `d(p)` elements and then takes `mx` – she wins.
If `d(p)` is **odd**, after the first player’s move the second player will
be the one who finally reaches `mx` – Alice loses.

Alice will of course choose the *best* occurrence of the maximum,
i.e. the one with the smallest distance to an end.
Only the parity of this minimal distance matters.

```

Every inner character `s[i]` (`1 ≤ i ≤ n‑2`) appears once as the right
character of pair `(i‑1,i)` and once as the left character of pair `(i,i+1)`;
its contributions cancel.
Only the first character contributes as a left character without a preceding
pair, and only the last character contributes as a right character without a
following pair. Hence the formula above. ∎



Consequences  

* `AB(s) = BA(s)` **iff** the first and the last character are equal.
* Changing any *inner* character does **not** influence the difference,
  because the formula depends only on the ends.
* If the ends are different, changing **one** of them makes them equal,
  and the difference becomes zero. One change is optimal (zero changes are
  impossible because the ends differ).

--------------------------------------------------------------------

#### 2.  Algorithm
For each test case  

```
if s[0] == s[-1]:
        answer = s                     # already balanced
else:
        # change the first character to the last one (any end works)
        answer = s[-1] + s[1:]
output answer
```

The algorithm performs at most one character modification, which is the
minimum possible.

--------------------------------------------------------------------

#### 3.  Correctness Proof  

We prove that the algorithm outputs a string with `AB = BA` using the
minimum possible number of changes.

---

**Lemma 1**  
If the first and last characters of a string are equal, then `AB = BA`.

*Proof.* Directly from the observation formula, the right‑hand side is `0`. ∎



**Lemma 2**  
If the first and last characters differ, any string obtained from the
original one with `AB = BA` must differ from the original in at least one
position.

*Proof.* By the observation, `AB = BA` requires the first and last
characters to be equal. Since they are different initially, at least one of
them must be changed. ∎



**Lemma 3**  
Changing exactly one of the two end characters makes the string balanced.

*Proof.* After changing, the first and last characters become equal,
hence by Lemma&nbsp;1 the new string satisfies `AB = BA`. ∎



**Theorem**  
For each test case the algorithm outputs a string with `AB = BA` and uses
the minimum possible number of character changes.

*Proof.*  
Two cases.

*Case 1:* `s[0] == s[-1]`.  
The algorithm outputs `s` unchanged. By Lemma&nbsp;1 it is already balanced,
so `0` changes are sufficient and obviously minimal.

*Case 2:* `s[0] != s[-1]`.  
The algorithm changes exactly the first character (one change) and outputs
the resulting string. By Lemma&nbsp;3 the output is balanced.
By Lemma&nbsp;2 any balanced string needs at least one change.
Thus the algorithm uses the minimum possible number (`1`). ∎



--------------------------------------------------------------------

#### 4.  Complexity Analysis  

For each test case we inspect the first and last character and possibly
build a new string of length `n`.  
Time   : `O(n)` (linear in the length of the string)  
Memory : `O(n)` only for the output string (the input itself is read anyway).

--------------------------------------------------------------------

        n = len(s)

        parent = list(range(n))

        def find(x):
            while parent[x] != x:
                parent[x] = parent[parent[x]]
                x = parent[x]
            return x

        def union(a, b):
            ra, rb = find(a), find(b)
            if ra != rb:
                parent[rb] = ra

                if find(i) == find(j):
                    possible = False

import sys

INF = 10**18

def min_time_one_dim(start, target, size):
    if size == 1:
        return 0 if start == target else INF
    period = 2 * (size - 1)
    s = start - 1
    t = target - 1
    # two possible positions in the unfolded line
    pos1 = t
    pos2 = period - t
    # time needed to reach each position modulo period
    time1 = (pos1 - s) % period
    time2 = (pos2 - s) % period
    return min(time1, time2)

def solve() -> None:
    data = sys.stdin.read().strip().split()
    if not data:
        return
    it = iter(data)
    t = int(next(it))
    out_lines = []
    for _ in range(t):
        n = int(next(it)); m = int(next(it))
        r_b = int(next(it)); c_b = int(next(it))
        r_d = int(next(it)); c_d = int(next(it))
        time_row = min_time_one_dim(r_b, r_d, n)
        time_col = min_time_one_dim(c_b, c_d, m)
        ans = min(time_row, time_col)
        out_lines.append(str(ans))
    sys.stdout.write("\n".join(out_lines))

if __name__ == "__main__":
